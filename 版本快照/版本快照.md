# 版本快照

> 记录每个 Sprint 测试工作的基线快照，用于版本对比、回归选取和用例复用

---

## 快照机制说明

### 作用

1. **版本基线**：固化每个 Sprint 结束时的需求理解、测试范围和用例规模，作为回归测试的选取依据
2. **变更追溯**：Sprint间对比快照，快速定位需求变更对已有用例的影响
3. **回归选取**：根据模块分布和优先级统计，为新 Sprint 的回归测试集提供参考
4. **交接凭据**：作为测试工作完成的书面凭据，记录交付物与实际文件的对应关系

### 生成时机

| 时机 | 说明 |
|------|------|
| **Sprint测试完成时**（主要） | 测试执行完毕、缺陷修复验证通过后，生成正式快照，数据为最终统计值 |
| **Sprint测试启动时**（可选） | 用例设计完成、尚未执行时，生成草稿快照，统计数据标注"待执行验证" |

### 快照包含内容

| 章节 | 说明 | 是否必填 |
|------|------|---------|
| 版本概述 | 版本号、迭代周期、需求文档路径（含权威来源）、快照生成日期 | 必填 |
| 产品目标 | 本Sprint的核心目标与成功标准 | 必填 |
| 需求摘要 | 功能需求编号/标题/概述，非目标清单 | 必填 |
| 测试用例统计 | 优先级统计（用例数+占比，事后统计，不设目标比例） | 必填 |
| 模块分布 | 各模块用例数 | 必填 |
| Bug统计 | 分优先级/严重程度/状态统计 | 已执行必填，未执行标注 待统计 |
| 风险评估 | 高影响风险点+缓解措施 | 必填 |
| 交付物清单 | 产出物名称+文件名+路径+状态 | 必填 |
| 设计方法分布 | EP/BVA/ST/EG占比 | 必填 |
| 关键决策记录 | 仅列出本Sprint中影响测试范围的核心决策（编号+结论），完整决策见知识库 | 可选 |
| 非目标清单 | 明确本期不做的能力及原因 | 必填 |
| 端到端验收场景 | 从用户视角的端到端操作链路及预期 | 可选 |

### 产品目标

| 编号 | 目标 | 成功标准 |
|------|------|----------|
| G1 | 租户资源启用边界清晰 | 平台给出租户可配置资源池；租户管理员可分别开启或关闭模型和公共接入点；关闭资源不可见、不可绑定、不可调用 |
| G2 | 接入点边界清晰 | 公共接入点由平台维护，业务接入点由租户维护；两类接入点的权限和路由规则不同 |
| G3 | API Key 可治理 | 大模型接入凭证独立管理；支持全部资源权限和部分资源权限；资源关闭后可追踪失效绑定 |
| G4 | 离职资源可清理 | 平台能识别离职人员创建的 API Key，并按租户策略禁用或删除 |
| G5 | 调用事实可追溯 | 调用明细事件进入 Kafka，ES 仅作为查询索引；调用日志、用量、告警和计费核对共享同一事实来源 |
| G6 | 用量、调用资格和限额可运营 | 普通用户、租户管理员、平台管理员看到不同范围的用量；租户调用前受 OM 点数包、试用或白名单资格控制；平台可对用户月度 Token 用量进行限额治理 |

### 需求摘要

#### 功能需求

| 编号 | 需求名称 | 所属模块 | 优先级 | 需求概述 |

| F3 | 大模型接入凭证重设计 | API Key | P0 | 大模型凭证独立管理；同一 Key 可绑定模型+接入点；支持全部资源权限（动态扩展）和部分资源权限（显式白名单） |

### 风险评估

| 风险点 | 影响等级 | 影响 | 缓解措施 |
|--------|---------|------|---------|
| 调用前置校验仅查本地状态 | 高 | OM/供应商状态的缓存延迟可能导致误放或误拦 | 明确最大延迟容忍窗口；缓存失效时安全侧处理（宁可误拦不误放） |
| API Key 状态同步到 Go 转发侧的可靠性 | 高 | 同步失败可能导致已禁用 Key 仍可调用 | 必须同步成功才算操作完成；异常时重试+告警；详见 ADR 0011 |
| Go 转发侧多 Key 自动禁用与接入点候选模型状态混淆 | 高 | 接入点候选模型不可用时可能误触发渠道级自动禁用 | 候选模型独立状态管理；禁用规则区分于上游 Key 禁用规则；详见技术边界说明 |
| OM 点数包=调用资格+额度 的语义耦合 | 中 | 点数到期后调用资格同时失效，可能影响业务连续性 | 明确到期前通知机制；续期后资格即时恢复 |
| Kafka 事件幂等与消费者扩展 | 中 | ES/用量/告警/OM消费者互不阻塞的前提是分区和消费组设计合理 | 提前确认分区数、Schema 管理、消费者组隔离策略 |
| 资源关闭失效绑定 v1 无手动清理入口 | 中 | 失效绑定累积可能影响 API Key 绑定列表可读性 | 同 providerId+modelId 重新启用时可自动恢复；v2 补充批量清理 |

| `detector` | ENUM('rule','qwen3guard','presidio','cloud_api') | 命中的检测层 | `qwen3guard` |
| `latency_ms` | INT | 本次检测耗时 | 28 |

**四级处置策略**：

| 风险等级 | 处置动作 | 典型场景 |
|---------|---------|---------|
| 高（high） | 拦截+安全代答+实时告警 | 明确注入攻击、已知越狱模板、高置信度违规 |
| 中（medium） | 放行+标记+异步告警（5分钟频控） | 疑似边界攻击、低置信度命中 |
| 低（low） | 放行+留痕 | 轻微敏感、仅需审计留档 |
| 无（none） | 直通 | 99%正常流量 |

**三模式风险容忍度**：

| 模式 | 启用检测层 | 适用场景 |
|------|-----------|---------|
| 严格 | 规则+Qwen3Guard+云审核 | 金融/政务/对外服务 |
| 常规（默认） | 规则+Qwen3Guard，按标签决定是否调云审核 | 企业内部业务 |
| 性能 | 仅规则 | 开发/测试 |
| 敏感租户（数据不出域） | 规则+Qwen3Guard（**强制跳过云审核**） | 涉密数据、本地化要求高的租户 |

**延迟预算**（总 < 100ms）：

| 环节 | 目标 | 说明 |
|------|------|------|
| 规则引擎 | < 10ms | 正则+AC自动机+关键词库 |
| 本地ML推理 | < 30ms | Qwen3Guard ONNX加速，CPU推理 |
| 云审核API | 20-40ms | 阿里云PLUS，按需触发 |
| 编排+网络 | 10-20ms | 并行调度开销 |
| **合计** | **< 80ms** | 预留20ms余量 |

**降级兜底策略**：

| 触发条件 | 降级动作 | 保障目标 |
|---------|---------|---------|
| 单层检测耗时 > 50ms | 本次该层跳过，按 `detector_timeout` 标记 | 主链路延迟可控 |
| 本地ML模型加载失败 | 降级为"仅规则层"，触发告警 | 至少规则拦截不中断 |
| 云审核API 5xx或超时 | 自动切换为本地兜底（仅规则+本地ML），事件标记 `fallback=true` | 可用性≥99.9% |
| 新规则/新模型上线 | 默认"观察模式"（仅打标不拦截），2周后手动切换 | 误杀率可控 |
| 规则热更新失败 | 回滚到上一版本，不影响在线流量 | 零停机迭代 |

| 4 | 管理员将注入检测切为拦截模式（二次确认） | 30秒内生效 |
