# 门禁检查清单（5道门禁）

> 每个阶段结束后，AI必须自动对照以下清单进行自检，并明确标注"通过/未通过"。
> 未通过项需列出具体问题和修正建议，并输出给用户确认。

---

## 门禁一：需求解读门禁

| # | 检查项 | AI自动检查 | 人工检查 |
|---|--------|-----------|---------|
| 1 | 核心业务目标清晰，一句话可概括 | ✅ | ✅ |
| 2 | 核心业务流程明确，主流程和关键分支无遗漏 | ❌ | ✅ |
| 3 | 测试范围明确，所有涉及模块已列出 | ✅ | ✅ |
| 4 | 不测试范围已明确，无模糊地带 | ✅ | ✅ |
| 5 | 高风险点已识别，≥3条 | ✅ | ✅ |
| 6 | 所有需求模糊点已列出并澄清 | ❌ | ✅ |
| 7 | 需求文档版本正确，与最新变更一致 | ✅ | ✅ |

**自检输出格式：**
```
## 需求解读门禁自检结果
- [通过/未通过] 检查项1：...
- [通过/未通过] 检查项2：...
...
整体结论：[全部通过 / 未通过，需修正以下问题]
未通过项及修正建议：
1. 检查项X：[具体问题描述] → 建议：[修正方案]
```

## 使用说明

- **AI自动检查（✅）**：AI可根据已生成内容自动核验
- **人工检查（✅）**：需要用户/测试人员结合实际业务判断
- **标注（❌）**：该检查项不适合或无法由AI/人工单独完成，需双方共同确认
- 所有AI自动检查项必须完成，人工检查项的结果在用户回复"门禁通过"时视为已确认
