### TC\_RES\_003



标题

：【#F1\_RES\_授权配置】正向：资源授权范围页配置模型授权+独立创建接入点+配置护栏



优先级

：P0



前置条件

：

> 2.模型库存在已纳管模型M;

### TC\_RES\_018



标题

：【#F1\_RES\_授权独立组合】正向：接入点授权与模型授权独立性四象限验证



优先级

：P1



前置条件

：

> 1. 公共接入点 ep-public（候选含模型 M）和模型 M 均已授权给 CL0100；
> 2. 普通用户 userA 拥有绑定 ep-public 的 API Key Key1 和绑定模型 M 的 API Key Key2



测试步骤

：

象限1（接入点已授权 + 模型已授权）：

1. 确认 ep-public 和模型 M 均在 CL0100 授权范围内；
2. userA 使用 Key1 调用 model=ep-public；
3. userA 使用 Key2 调用 model=glm-5.1
   象限2（接入点已授权 + 模型未授权）：
4. 平台管理员撤销 CL0100 对模型 M 的授权；
5. userA 使用 Key1 调用 model=ep-public（接入点内部路由仍命中模型 M，不二次校验模型授权）；
6. userA 使用 Key2 调用 model=glm-5.1（应被拒绝）
   象限3（接入点未授权 + 模型已授权）：
7. 平台管理员重新授权模型 M，撤销 CL0100 对 ep-public 的授权；
8. userA 使用 Key1 调用 model=ep-public（应被拒绝，展示「平台撤销授权」）；
9. userA 使用 Key2 调用 model=glm-5.1（应成功）
   象限4（接入点未授权 + 模型未授权）：
10. 平台管理员撤销 CL0100 对模型 M 的授权；
11. userA 使用 Key1 调用 model=ep-public（应被拒绝）；
12. userA 使用 Key2 调用 model=glm-5.1（应被拒绝）



预期结果

：

- 象限1：Key1 和 Key2 调用均成功；
- 象限2：Key1 调用成功（ep-\* 调用只校验接入点授权，内部路由候选模型不过授权层），Key2 调用被拒绝；
- 象限3：Key1 调用被拒绝（接入点撤销授权 → 绑定失效），Key2 调用成功；
- 象限4：Key1 和 Key2 调用均被拒绝
