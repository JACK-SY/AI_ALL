# 治理设置

| 属性 | 值 |
|------|-----|
| 一级菜单 | 治理设置 |
| 优先级 | P0（离职资源清理）/ P1（通知联系人、调用资格/OM点数包） |
| 主要用户 | 租户管理员、平台管理员 |
| 需求来源 | F7 离职资源清理、F6 用量视角与调用资格与用量限额 |

## 模块定位与职责

治理设置是租户和平台管理员的**运营治理中心**，包含三个子页面：离职资源清理、通知联系人、调用资格/OM点数包。它负责处理人员变动后的资源清理、通知触达配置、以及调用资格和点数余额的监控。

核心职责：确保模型调用全生命周期的治理闭环——人走了资源清、出事了有人知、没钱了不能用。

## 核心业务规则

### 离职资源清理规则（F7）

1. 离职人员由 IAM 识别，平台不自行判定
2. 通过定时任务同步离职人员名单（接口 `POST /CRM_WebApi/api/GetCMSMV`，`Isresign=N`）
3. 离职治理能力受**部署级离职治理启用配置**控制；关闭时不启动同步、不展示入口、不访问相关表
4. API Key 是否属于离职资源以**创建人**为准
5. 默认策略：**禁用**离职人员创建的 API Key
6. 特定 AI Coding 租户可配置为**自动删除**（平台管理员维护白名单）
7. 离职清理是凭证级处置，**不产生**资源关闭失效绑定
8. 租户管理员可一键删除待处理 API Key
9. 自动和手动处置均进入审计日志
10. 本期**不发送真实离职通知**，仅保留通知状态和页面提示

### 用户级月度 Token 限额规则（F6 短解）

1. 限额主体是用户，不是单把 API Key
2. 限额归属用户沿用 API Key 创建人
3. 只有进入**限额租户白名单**的租户启用限额治理
4. 已配置用户按配置额度阻断；未配置用户默认不限额
5. 额度为 0 表示显式禁止；删除配置回到未配置规则
6. 平台管理员通过后台配置表或运维方式维护
7. 按自然月统计；新月归零，配置沿用
8. 达限额时前置拒绝，返回不可重试失败
9. 允许最后一次放行调用轻微超过限额

### OM 计量豁免规则

1. 按 `x-billing-TenantId` 判断 OM 计量豁免（长期契约）
2. 豁免在模型平台侧丢弃 OM 支线，不进入 OM 扣减链路
3. 豁免只替代商业计量校验，不替代权限和治理校验
4. 豁免名单用 MySQL 配置表维护，主链路只从 Redis 读取
5. Redis 未同步时不兜底查 MySQL
6. Redis 异常时默认不豁免

## 状态流转

### 离职资源清理状态

```
正常 → 离职待处理 → 已禁用待删除/自动删除中 → 已删除/处置失败
```

### 调用资格状态

```
有资格（OM点数包/试用/白名单）↔ 无资格
```

## 与其他模块的上下游关系

| 方向 | 关联模块 | 关系说明 |
|------|---------|---------|
| 上游 | 接入凭证 | 离职清理禁用/删除 API Key |
| 上游 | 资源管理 | 资源关闭影响调用资格 |

| 上游 | OM 系统 | 提供点数包状态和余额 |
| 下游 | 调用观测 | 限额超额事件写入调用明细 |
| 下游 | 接入凭证 | 同步禁用/删除状态到 Go 转发侧 |
