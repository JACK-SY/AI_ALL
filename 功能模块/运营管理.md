## 模块定位与职责

运营管理是平台管理员的**运营配置与监控中心**，包含模型配置和租户用量监控两个子页面。模型配置负责按模型 ID 配置计价规则，租户用量监控提供跨租户维度的用量统计与排行。

核心职责：实现平台级计价配置闭环和全租户用量可观测性。

| 概念 | 定义 | 推荐用词 | 避免用词 |

## 核心业务规则

### 模型配置规则

1. 按模型 ID 配置计价规则，一个模型 ID 只能有一条生效配置
2. 支持两种计费方式：按量计费（输入单价 + 输出单价）、阶梯计费（多档位递进定价）
3. 阶梯计费必须至少配置 2 个档位
4. 未配置计价规则的模型显示"未配置"状态，不影响模型调用
5. 配置变更实时生效，无需审批
6. 概览区域展示统计：已纳管模型数、已配置模型数、未配置模型数

### 租户用量监控规则

1. 仅平台管理员可见，支持按租户名称和时间范围筛选
2. 统计数据包含所有租户的汇总，覆盖租户调用排行和用户调用排行
3. 调用日志明细比模型观测（租户侧）多出 User-Agent 和 IP 两个字段
4. 与模型观测（租户侧）共享底层 Kafka 事实流，但视角为跨租户全量

## 与其他模块的上下游关系

| 方向 | 关联模块 | 关系说明 |
|------|---------|---------|
| 上游 | 模型管理 | 模型配置的模型列表来源于模型库的已纳管模型 |
| 下游 | 模型观测 | 模型配置的计价信息可关联到租户侧的用量统计；租户用量监控与模型观测共享调用日志底层数据 |
| 上游 | IAM | 平台管理员身份校验 |
