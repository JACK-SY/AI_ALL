# 资源管理

| 属性 | 值 |
|------|-----|
| 一级菜单 | 资源管理 |
| 优先级 | P0 |
| 主要用户 | 租户管理员 |
| 需求来源 | F1 平台资源池与租户资源管理 |

## 模块定位与职责

资源管理是租户管理员的**资源开关控制台**，用于在平台资源池范围内开启或关闭本租户可用的模型和公共接入点。它本身**不维护**模型资产（那是模型库的职责），也不编辑公共接入点内部路由（那是接入点管理的职责）。

核心职责：决定"本租户能用什么"——在平台给出的可选范围内，由租户管理员做最终启用/关闭决策。

## 核心概念

| 概念 | 定义 | 推荐用词 | 避免用词 |
|------|------|---------|---------|
| 平台资源池 | 平台管理员维护的租户可分配资源集合 | 平台资源池 | 全平台资源任意开通 |
| 租户可选模型池 | 平台预先分配给租户的模型集合 | 租户可选模型池 | 租户自助绑定全平台模型 |
| 租户资源启用配置 | 租户管理员在资源池内开启或关闭资源的配置 | 租户资源启用配置 | 平台指定租户发布、用户级申请审批 |
| 资源关闭失效绑定 | 资源关闭后 API Key 对应绑定保留但调用立即失效 | 资源关闭失效绑定 | 关闭资源时删除 API Key 绑定 |
| 默认启用策略 | 资源首次进入租户资源池时的默认启用状态 | 默认启用策略 | 平台加入资源池后一律自动开放 |

## 角色权限矩阵

| 页面 | 普通用户 | 租户管理员 | 平台管理员 |

## 核心业务规则

1. 平台资源池限制租户管理员可配置范围；租户管理员不能任意开启全平台所有模型或接入点
2. 租户可选模型池由平台预先分配；首版可由该租户可用公共接入点绑定的候选模型列表形成
3. 模型和公共接入点启用状态独立配置
4. 关闭公共接入点只影响该公共接入点在本租户下的可见、绑定和调用，不影响背后候选模型
5. 关闭模型只影响模型广场可见、模型级 API Key 绑定、模型直调和业务接入点新增候选，不影响已开启公共接入点的内部路由
6. 业务接入点创建或编辑时只能选择本租户已启用模型
7. 租户管理员关闭某个模型后，业务接入点中已有该模型候选进入不可用或异常状态；系统不自动替换成其他模型
8. 关闭模型或公共接入点后，已有 API Key 绑定保留但进入资源关闭失效绑定状态
9. 重新开启同一资源后，历史失效绑定可自动恢复有效；不同资源不得自动复用旧绑定
10. 资源关闭、开启、默认启用策略和 API Key 失效绑定变化都需记录审计日志

## 状态流转

### 模型资源状态

```
平台已分配（默认启用）←→ 租户已关闭
平台已分配（默认关闭）←→ 租户已启用
```

### API Key 绑定受资源状态影响

```
资源已启用 → 绑定有效 → 可正常调用
资源已关闭 → 绑定进入"资源关闭失效绑定"状态 → 调用立即失败
资源重新启用 → 失效绑定自动恢复有效
```

## 与其他模块的上下游关系

| 方向 | 关联模块 | 关系说明 |
|------|---------|---------|
| 上游 | 模型管理 → 模型库 | 模型资产来源，决定了资源池有哪些模型可供分配 |
| 上游 | 模型管理 → 资源池默认启用策略 | 平台管理员配置默认启用/关闭 |
| 上游 | 接入点管理 | 定义了公共接入点，决定哪些接入点可加入资源池 |
| 下游 | 模型广场 | 过滤条件：只有租户已启用资源才在模型广场展示 |
| 下游 | 接入凭证 | 资源关闭产生失效绑定，影响 API Key 权限 |
| 下游 | 业务接入点 | 业务接入点只能选择租户已启用模型作为候选 |

## 关键决策编号及结论

| 决策编号 | 结论 | 影响 |
|---------|------|------|
| D-01 | 租户可用范围分两层 | 资源管理只管第二层开关 |
| D-02 | 平台资源池含公共接入点和租户可选模型池 | 两种资源独立开关 |
| D-03 | 租户级自动可用，不做用户级开通 | 不提供用户级开通流程 |
| D-04 | 本期不做自助接入、申请审批等 | 资源管理不包含审批功能 |
| D-04i | 公共接入点和模型是独立资源开关 | 关闭不联动 |
| D-04j | 平台可设置默认启用策略 | 策略只影响首次进入资源池 |
| D-09 | 调用日志命中模型不代表模型直接启用 | 用量统计可聚合但不可跳转 |
