# 接入点管理

| 属性 | 值 |
|------|-----|
| 一级菜单 | 接入点（接入点管理） |
| 优先级 | P0 |
| 主要用户 | 平台管理员、租户管理员 |
| 需求来源 | F2 接入点归属类型与路由意图 |

## 模块定位与职责

接入点管理是**模型调用的路由中枢**，负责维护公共接入点和业务接入点，配置路由意图与路由策略。它是"业务怎么调用"的控制中心。

核心职责：决定"调用目标指向哪里"和"路由怎么分发"。

## 核心概念

| 概念 | 定义 | 推荐用词 | 避免用词 |
|------|------|---------|---------|
| 接入点 | 模型调用的统一入口标识，以 ep- 前缀表示 | 接入点 | 端点、模型别名 |
| 公共接入点 | 平台管理员维护的多租户共享调用入口 | 公共接入点 | 公开接入点 |
| 业务接入点 | 租户管理员维护的单租户专用调用入口 | 业务接入点 | — |
| 接入点路由意图 | 说明候选模型为什么被放在同一接入点下 | 接入点路由意图 | 接入点类型（表达路由意图时） |
| 负载均衡路由意图 | 同一模型能力等价分流 | 负载均衡路由意图 | 不同模型混合分流 |
| 健康降级路由意图 | 主模型不可用时切换到备用模型 | 健康降级路由意图 | 负载均衡（表达主备降级时） |
| 复杂度路由意图 | 调用方显式传入任务复杂度标记 | 复杂度路由意图 | 隐式模型选择 |
| 顺序路由 | 按候选顺序依次尝试 | 顺序路由 | 主备（单独使用） |
| 权重路由 | 按权重比例分配流量 | 权重路由 | 随机路由（表达等价分流时） |
| 接入点候选模型绑定 | 接入点与纳管模型的路由关联 | 接入点候选模型绑定 | 自动切换到另一个模型 |
| AI安全护栏 | 接入点级别的安全检测与拦截策略 | AI安全护栏 | 安全围栏（旧称） |
| 推荐配置 | 平台提供的安全护栏推荐模板 | 推荐配置 | 默认策略 |

## 角色权限矩阵

| 页面 | 普通用户 | 租户管理员 | 平台管理员 |
|------|---------|-----------|-----------|
| 接入点列表 | 可见本租户已启用接入点 | 可见本租户的业务接入点和已启用公共接入点 | 可见全平台接入点 |

## 核心业务规则

### 归属类型规则

1. 公共接入点由平台管理员创建和维护，面向多租户，租户侧只读
2. 业务接入点由租户管理员创建和维护，归属当前租户
3. 业务接入点只能选择本租户已启用模型作为候选

### 路由意图规则

1. 首版仅在公共接入点配置路由意图；业务接入点不开放
2. 公共接入点必须配置路由意图：负载均衡/健康降级/复杂度路由
3. 负载均衡意图下候选由平台管理员人工确认为同一模型能力
4. 负载均衡首版支持随机和轮询，默认随机
5. 轮询需转发侧具备可维护游标缓存能力；不满足时不允许配置或须提示
6. 健康降级必须明确主模型、降级模型和降级顺序
7. 复杂度路由支持 `X-Task-Complexity` Header 传入 `simple`/`normal`/`complex` 三档，每档选一个模型；缺少走默认 normal；非法按参数错误拒绝

### 失败切换规则

1. **可重试失败**触发切换：上游超时、连接失败、5xx、429 供应商限流、模型临时不可用
2. **不可重试失败**不触发切换：权限不足、参数错误、余额不足、AI 安全护栏拦截、合规拦截、模型拒答
3. 实际命中模型、路由意图、是否降级、复杂度请求/解析/默认值必须写入调用明细事件

### 租户侧展示规则

1. 公共接入点租户侧只读展示调用方式、状态、AI 安全护栏状态、路由意图和当前默认模型摘要
2. 不展示完整候选列表、候选数量、轮询游标、供应商配置明细
3. 复杂度路由展示允许的复杂度标记、默认档位和每档用途
4. 健康降级展示"可能降级"提示
5. 业务接入点候选选择器只展示 modelId + providerName

### AI 安全护栏规则（F8）

1. 接入点可绑定一条接入点策略
2. 新建接入点默认不启用 AI 安全护栏
3. 页面引导用户套用推荐配置
4. 首次套用后默认观察模式
5. 可从观察模式切换到拦截模式
6. 观察模式只记录风险结果，不阻断调用
7. 拦截模式命中高风险规则时阻断调用并返回安全响应
8. 检测范围：内容合规、敏感内容、提示词攻击、敏感信息泄露
9. 检测结果写入调用明细事件

### 候选模型异常规则

1. 业务接入点已有候选被关闭或不可用后该候选进入不可用或失效状态
2. 系统不自动替换成其他模型；租户管理员需手动编辑
3. 所有候选失效时调用失败，提示需编辑路由候选
4. 公共接入点候选异常由平台管理员处理
5. 本期不做业务接入点候选异常的主动通知

## 状态流转

### 接入点状态

```
创建 → 启用 → 禁用
```

### 候选模型状态

```
正常 → 不可用（模型关闭/下架/供应商变更）→ 重新绑定 → 正常
```

### AI 安全护栏策略状态

```
未启用 → 观察模式 → 拦截模式
```

## 与其他模块的上下游关系

| 方向 | 关联模块 | 关系说明 |
|------|---------|---------|
| 上游 | 模型管理 | 提供候选模型数据（纳管模型列表） |
| 下游 | 模型广场 | 接入点信息展示在模型详情的"接入方式"页签 |
| 下游 | 基础设置 | 接入点是 API Key 绑定目标之一；候选异常不影响接入点绑定状态 |
| 下游 | 模型观测 | 调用日志记录实际命中模型和路由意图；用量统计支持按接入点聚合 |

## 关键决策编号及结论

| 决策编号 | 结论 | 影响 |
|---------|------|------|
| D-04e | 接入点保留为一级菜单 | 不合并进资源管理 |
| D-05 | 公共接入点独立进入资源池 | 不要求同时启用背后候选模型 |
| D-06 | 公共接入点启用不扩大模型启用范围 | 模型可见性独立控制 |
| D-08 | 租户侧只展示路由摘要 | 完整路由配置仅平台管理员可见 |
| D-13a | 接入点区分归属类型和路由意图 | 详见 ADR 0005 |
| D-29 | 策略粒度以接入点策略为主 | 平台不自动套用策略 |
| D-31a | MODEL_ID 变更不自动切换候选 | 需手动重新绑定 |
| D-31b | 业务接入点候选异常不主动通知 | 租户管理员需自查 |
