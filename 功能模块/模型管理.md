## 模块定位与职责

模型管理是平台管理员的**模型资产维护中心**，负责维护模型库的完整信息、管理模型供应商和纳管模型实例。它是模型广场和基础设置的数据源头。

核心职责：决定"平台有什么模型"和"模型怎么分配给租户"。

## 核心业务规则

### 模型发布流程（F9）

1. 平台管理员确认纳管模型信息（模型ID、供应商来源、能力参数、健康状态）
2. 选择模型进入哪些租户的平台资源池，设置默认启用状态
3. 选择是否生成公共接入点（默认不勾选）；如勾选→配置接入点名称、路由策略、状态、加入租户资源池范围和默认启用状态
4. 可选套用 AI 安全护栏推荐配置（默认不启用，启用后默认观察模式）
5. 提交→资源进入对应租户的平台资源池→是否立刻对普通用户可见取决于默认启用策略和租户管理员后续配置

### 默认启用策略规则

1. 纳管模型发布前必须明确是否加入租户平台资源池
2. 加入时必须设置默认启用状态
3. 默认启用策略只影响资源首次进入租户资源池，不覆盖租户管理员手动配置
4. 生成公共接入点必须由平台管理员显式确认
5. 未勾选不自动创建接入点
6. 公共接入点生成后可独立加入或移出租户资源池
7. 公共接入点进入租户资源池不要求租户同时启用背后候选模型

### MODEL_ID 变更影响治理规则（F11）

1. 接入点候选模型绑定以 `providerId + modelId` 为准，不因供应商或渠道配置变更自动切换
2. 编辑 MODEL_ID、模型列表、模型映射、上游地址前，系统应提供影响分析
3. 影响分析必须覆盖公共接入点和业务接入点
4. 影响分析至少展示：受影响接入点数量、ID、名称、类型、候选位置、路由角色、优先级、权重、归属租户、当前状态
5. 候选变更后不可用时接入点进入需重新绑定或异常状态；不产生 API Key 资源关闭失效绑定
6. 同一候选后续恢复可用时接入点可继续调用，不要求重建 API Key
7. 切换供应商/模型/渠道必须编辑接入点路由候选重新同步
8. 接入点编辑页必须支持将受影响候选直接替换为新的 `providerId + modelId`
9. 供应商/渠道变更确认页只提示影响范围，不提供批量重绑入口
10. 直接重新绑定时应保留候选在路由中的位置、角色、优先级和权重
11. 公共接入点由平台管理员处理影响；业务接入点让创建租户管理员看到候选异常
12. 本期不做业务接入点候选异常的主动通知
13. 变更、影响确认、候选异常和恢复均应记录审计日志

## 状态流转

### 纳管模型生命周期

```
注册 → 上架 → 分配到资源池 → 租户启用/关闭 → 下架
```

### 供应商变更影响流转

```
正常候选 → 编辑触发影响分析 → 确认保存 → 候选不可用（需重新绑定/异常） → 人工重新绑定 → 同步转发侧 → 恢复可用
```

## 与其他模块的上下游关系

| 方向 | 关联模块 | 关系说明 |
|------|---------|---------|
| 下游 | 模型广场 | 提供模型资产数据，决定哪些模型可以被展示 |
| 下游 | 基础设置 | 模型上下架影响 API Key 绑定有效性；供应 MODEL_ID 变更触发影响分析 |
| 下游 | 接入点管理 | 提供候选模型数据，公共接入点绑定的候选模型来源于模型库 |
| 双向 | 模型观测 | 模型相关信息写入调用明细事件；用量统计可按模型维度聚合 |
