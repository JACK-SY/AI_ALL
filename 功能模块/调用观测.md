# 调用观测

| 属性 | 值 |
|------|-----|
| 一级菜单 | 调用观测 |
| 优先级 | P0 |
| 主要用户 | 普通用户、租户管理员、平台管理员 |
| 需求来源 | F5 调用日志事实流、F6 用量视角与调用资格与用量限额 |

## 模块定位与职责

调用观测是模型调用的**可观测性中心**，由调用日志和用量统计两个子页面组成。调用日志基于 Kafka 事实流提供全链路调用明细查询，用量统计提供多维度用量聚合分析。两者共同支撑排障、审计、成本治理和计费核对。

核心职责：让每一次模型调用"看得见、查得到、算得清"。

## 核心概念

| 概念 | 定义 | 推荐用词 | 避免用词 |
|------|------|---------|---------|
| 调用明细事件 | 一次模型调用尝试的不可变事实记录 | — | — |

| 实际命中模型 | 接入点路由实际到达的纳管模型 | — | — |
| 入口无关成本限额 | 无论直接调用还是接入点调用，命中平台成本模型就计入同一限额 | 入口无关成本限额 | 接入点专属限额、模型直调专属限额 |

## 核心业务规则

### 调用日志规则（F5）

1. 每次调用尝试都必须有可追踪记录
2. 调用明细事件优先写入 Kafka，message key 为 `tenant_id:api_key_id`
3. 事件使用 `event_id` / `request_id` 做幂等
4. ES 消费者写入月度索引 `call_log_YYYYMM`
5. 用量消费者更新统计表
6. 告警消费者处理异常和安全护栏提醒
7. OM 计量消费者扣除或扣点
8. 前置校验失败（无资格、余额不足、Key无效）不请求上游但仍在日志中记录
9. 实际命中模型、路由意图、是否降级、复杂度信息必须写入事件

### 用量视角规则（F6）

1. 普通用户默认"我的用量"，不可切换
2. 租户管理员默认"本租户用量"，可切换到"我的用量"
3. 平台管理员默认"全平台用量"，可切换到指定租户或"我的用量"
4. 公共接入点命中租户未直接启用模型时，用量统计仍可按实际命中模型筛选和聚合
5. 未启用模型筛选项不提供模型详情跳转、绑定或创建 API Key

### 调用资格规则（F6）

1. 具备以下任一条件视为有调用资格：有效 OM 点数包、试用期、调用白名单
2. 无资格时禁止调用生产模型
3. 点数不足时调用前置校验失败
4. OM 状态不在每次调用时远程查询，通过事件/定时任务同步到本地
5. OM 消费失败不回滚已返回的响应，后续通过重试/对账/运营处理

## 状态流转

### 调用事件状态

```
请求到达 → 前置校验 → 路由判定 → 上游调用 → 响应返回 → 事件写入Kafka → 消费入库
                           ↓
                    校验失败（仍记录事件）
```

## 与其他模块的上下游关系

| 方向 | 关联模块 | 关系说明 |
|------|---------|---------|
| 上游 | 接入凭证 | API Key 是日志和统计的核心维度 |
| 上游 | 接入点管理 | 接入点是路由和降级信息的来源 |
| 上游 | 资源管理 | 资源启用状态影响调用资格前置校验 |
| 上游 | 治理设置 | OM 点数包/资格状态影响调用结果 |
| 下游 | 模型广场 | 用量统计中的模型维度聚合结果 |
| 下游 | 模型管理 | 日志中命中模型的供应商归属信息 |

## 关键决策编号及结论

| 决策编号 | 结论 | 影响 |
|---------|------|------|
| D-09 | 调用日志命中模型不扩大模型级授权范围 | 未启用模型不提供跳转 |
| D-13 | 用量统计可按实际命中模型聚合 | 即使租户未启用该模型 |
| D-32 | 调用前置校验只查本地状态 | 安全侧处理缓存延迟 |
| D-33 | 调用明细事件优先进入 Kafka | 事件幂等是关键 |
