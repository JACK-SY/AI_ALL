## 模块定位与职责

基础设置是开发者的**调用身份凭据管理入口**，核心子页面 API Key 管理负责颁发与验证调用身份、控制权限范围、管理与资源关闭/人员离职的联动失效。

核心职责：决定"谁在调用"和"能调用什么"。

## 核心概念

| 概念 | 定义 | 推荐用词 | 避免用词 |
|------|------|---------|---------|
| API Key | 大模型调用的身份凭据，归属租户和真实创建人 | API Key | 密钥（单独使用） |
| 全部资源权限 | 对归属租户当前和未来已启用资源动态生效 | 全部资源权限 | — |
| 部分资源权限 | 只对显式选择的模型和接入点生效（白名单） | 部分资源权限 | — |
| 资源关闭失效绑定 | 资源关闭后 API Key 对应绑定保留但调用立即失效 | 资源关闭失效绑定 | 关闭资源时删除 API Key 绑定 |
| 失效绑定恢复 | 同一资源重新启用后历史失效绑定可自动恢复 | — | 不同资源自动复用旧绑定 |
| 转发侧租户账号 | Go 转发侧与模型平台租户一一映射的技术账号 | 转发侧租户账号 | 转发侧用户（未说明时） |
| 限额归属用户 | 用户级限额归属沿用 API Key 创建人 | 限额归属用户 | 终端使用人（短期方案） |

### API Key 详情

| 区域 | 字段 |
|------|------|

| 审计记录 | 所有状态变更和权限变更记录 |

## 核心业务规则

### 权限范围规则

1. 同一把 API Key 可同时绑定模型和接入点
2. 模型绑定按具体纳管模型选择，后台按 `providerId + modelId` 保存和校验，页面展示 `modelId` + `providerName`
3. 全部资源权限：对归属租户当前和未来已启用模型、接入点动态生效；新增启用资源后不需要逐把更新；不绕过资源启用状态、OM 调用资格、余额和 AI 安全护栏
4. 部分资源权限：只对显式选择的模型和接入点生效；新增启用资源不自动进入
5. 调用目标以 `ep-*` 开头时按接入点校验，否则按模型校验
6. 调用接入点时不要求 API Key 同时启用或绑定背后候选模型

### 资源关闭失效绑定规则

1. 绑定的模型或公共接入点被租户管理员关闭后，该资源绑定进入资源关闭失效绑定
2. 绑定记录保留用于排查，但对应目标调用立即失败
3. 模型绑定失效时失效对象展示为模型
4. API Key 绑定接入点时，接入点背后候选不可用不改变接入点绑定状态，也不产生资源关闭失效绑定
5. 同一 `providerId + modelId` 重新启用并满足条件时，资源关闭失效绑定可自动恢复有效，需同步 Go token 并清理 Redis
6. 若恢复的是不同 `providerId` 或不同 `modelId`，旧失效绑定不得自动恢复

### 转发侧同步规则（F4）

1. 租户首次创建大模型接入凭证时按需创建转发侧租户账号
2. 模型平台保存 `tenant_id → relay_user_id` 映射
3. 同租户后续 API Key 复用同一个 `relay_user_id`
4. API Key 状态或授权变化后模型平台主动同步 Go 转发侧 token
5. 同步完成或失败后都要更新模型平台侧同步状态
6. 同步后主动清理 Redis 缓存，不等待自然过期
7. 全部资源权限的 API Key 在 Go 侧不得固化模型或接入点白名单

### 同步触发场景

| 触发 | 动作 |
|------|------|
| 创建 API Key | 创建或复用转发侧租户账号，创建 Go token |
| 禁用 API Key | Go token 置为不可用，清理缓存 |
| 启用 API Key | Go token 置为可用，清理缓存 |
| 删除 API Key | 删除或禁用 Go token，清理缓存 |
| 权限范围变化 | 更新 Go token 有效权限，清理缓存 |
| 资源关闭失效绑定产生 | 从 Go token 有效权限中移除对应目标，清理缓存 |
| 资源关闭失效绑定恢复 | 将对应目标加入 Go token 有效权限，清理缓存 |
| 离职资源清理 | 同步禁用或删除状态，清理缓存 |

## 状态流转

### API Key 状态

```
创建 → 生效中 ↔ 已禁用 → 已删除
          ↓
       同步中 → 同步成功/同步失败
```

### 绑定状态

```
有效绑定 → 资源关闭失效绑定 → 自动恢复有效/手动重新绑定
```

## 与其他模块的上下游关系

| 方向 | 关联模块 | 关系说明 |
|------|---------|---------|
| 上游 | 模型管理 | 提供纳管模型数据用于绑定选择；模型上下架影响绑定有效性 |
| 上游 | 接入点管理 | 接入点是绑定目标之一；候选异常不改变接入点绑定状态 |
| 上游 | 治理设置 | 离职清理触发 API Key 禁用/删除 |
| 下游 | 模型观测 | API Key 是调用日志和用量统计的维度之一 |
| 双向 | 网关转发 | Go 转发侧是 API Key 校验和权限校验的执行端 |

## 关键决策编号及结论

| 决策编号 | 结论 | 影响 |
|---------|------|------|
| D-14 | 同一把 API Key 可同时绑定模型和接入点 | 权限校验需判断目标类型 |
| D-15 | 调用归属主体以 API Key 创建人为准 | 日志/用量/权限/审计以此为准 |
| D-16 | 全部资源权限动态扩展 | 新增资源不需逐把更新 |
| D-17 | 部分资源权限显式白名单 | 新增资源不自动进入 |
| D-18 | 资源关闭绑定保留但调用失效 | 需展示失效状态 |
| D-19 | 失效绑定必须同步 Go 转发侧 | 同步是闭环必要步骤 |
| D-20 | 失效只影响对应目标 | 同一 API Key 其他绑定继续可用 |
| D-21 | v1 不提供手动重新生效入口 | 自动恢复仅限同一资源 |
| D-26 | 离职清理不产生资源关闭失效绑定 | 两种联动机制独立 |
