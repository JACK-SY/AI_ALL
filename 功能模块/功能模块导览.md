## 模块总览

平台共设 **7个一级菜单**，按职责分为四个层级：**资产层 → 控制层 → 凭证层 → 观测层**，另有**治理层**横跨多个层级提供治理闭环。

| 层级 | 模块 | 核心职责 | 详情 |
|------|------|---------|------|
| 🏛 资产层 | [模型管理](模型管理.md) | 决定"平台有什么模型"和"模型怎么分配" | 平台管理员维护模型库、配置默认启用策略、治理MODEL_ID变更 |
| 🔧 控制层 | [资源管理](资源管理.md) | 决定"本租户能用什么" | 租户管理员在平台资源池范围内开启/关闭模型和接入点 |
| 🔧 控制层 | [接入点管理](接入点管理.md) | 决定"业务怎么调用"和"路由怎么分发" | 平台管理员维护公共接入点，租户管理员维护业务接入点 |
| 👁 展示层 | [模型广场](模型广场.md) | 面向用户的"模型发现入口" | 展示经过资源启用过滤后的可用模型，不维护资产 |
| 🔑 凭证层 | [接入凭证](接入凭证.md) | 决定"谁在调用"和"能调用什么" | 管理API Key身份凭据、权限绑定、失效联动 |
| 📊 观测层 | [调用观测](调用观测.md) | 让每次调用"看得见、查得到、算得清" | 调用日志全链路明细 + 多维度用量统计 |
| ⚖ 治理层 | [治理设置](治理设置.md) | 人走了资源清、出事了有人知、没钱了不能用 | 离职清理、通知联系人、调用资格/OM点数包 |

## 核心业务流与涉及的模块

### 业务流①：模型上架 → 对用户可见

```
模型管理（注册/上架）
    → 资源管理（分配到租户资源池，默认启用策略生效）
        → 模型广场（租户已启用模型对用户可见）
```

### 业务流②：业务系统调用模型

```
接入凭证（创建API Key，绑定模型/接入点）
    → 接入点管理（通过接入点路由到候选模型）
        → 调用观测（调用日志记录 + 用量统计）
```

### 业务流③：资源关闭生效链

```
资源管理（租户管理员关闭模型/接入点）
    → 接入凭证（已有绑定进入"资源关闭失效绑定"状态，调用立即失败）
        → 转发侧同步（从Go token有效权限中移除，清理Redis）
```

### 业务流④：离职人员清理

```
治理设置（识别离职人员创建的API Key）
    → 接入凭证（默认禁用 / AI Coding租户自动删除）
        → 转发侧同步（同步禁用/删除状态，清理Redis）
```

> 注意：离职清理**不产生**资源关闭失效绑定，两种联动机制独立。

## 角色权限速查

| 模块 | 普通用户 | 租户管理员 | 平台管理员 |
|------|---------|-----------|-----------|
| [模型管理](模型管理.md) | 不可见或只读 | 不可见或只读 | **完全控制** |
| [资源管理](资源管理.md) | 不可见 | **可见、可操作** | 按租户查看 |
| [接入点管理](接入点管理.md) | 可见已启用接入点（只读） | 编辑业务接入点 | 编辑公共接入点 |
| [模型广场](模型广场.md) | 可见本租户已启用模型 | 额外可见启用状态 | 可见全平台模型 |
| [接入凭证](接入凭证.md) | 可见本人创建的凭证 | 可见本租户凭证 | 可见全平台凭证 |
| [调用观测](调用观测.md) | 本人调用/我的用量 | 本租户调用/用量 | 全平台调用/用量 |
| [治理设置](治理设置.md) | 不可见 | 本租户治理配置 | 全平台治理配置 |

## 关键术语速查

| 术语 | 定义 | 所属模块 |
|------|------|---------|
| 纳管模型 | 平台正式注册并在模型库中维护的模型 | 模型管理 |
| 平台资源池 | 平台管理员维护的租户可分配资源集合 | 资源管理 |
| 租户资源启用配置 | 租户管理员在资源池内开启/关闭资源的配置 | 资源管理 |
| 资源关闭失效绑定 | 资源关闭后绑定保留但调用立即失效 | 接入凭证 |
| 公共接入点 | 平台管理员维护的多租户共享调用入口 | 接入点管理 |
| 业务接入点 | 租户管理员维护的单租户专用调用入口 | 接入点管理 |
| 接入点路由意图 | 说明候选模型为什么被放在同一接入点下 | 接入点管理 |
| AI安全护栏 | 接入点级别的安全检测与拦截策略 | 接入点管理 |
| 全部资源权限 | API Key对租户当前和未来已启用资源动态生效 | 接入凭证 |
| 部分资源权限 | API Key只对显式选择的模型和接入点生效 | 接入凭证 |
| 调用明细事件 | 一次模型调用尝试的不可变事实记录 | 调用观测 |
| 调用资格 | 租户是否能调用生产模型的本地判断依据 | 治理设置 |
| OM点数包 | 同时代表调用资格和点数额度的购买包 | 治理设置 |

> 完整术语定义参见：[项目概览/model-ops-knowledge/CONTEXT.md](../项目概览/model-ops-knowledge/CONTEXT.md)
