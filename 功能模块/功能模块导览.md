## 模块总览

平台共设 **9个一级菜单**，分为 **三大类**：核心运控模块、工具组模块（仅目录占位）、运营管理模块。

### 核心运控模块（6个）

按职责分为四个层级：**资产层 → 控制层 → 凭证层 → 观测层**，另有**治理层**横跨多个层级提供治理闭环。

| 层级 | 模块 | 核心职责 | 详情 |
|------|------|---------|------|
| 🏛 资产层 | [模型管理](模型管理.md) | 决定"平台有什么模型"和"模型怎么分配" | 管理模型库、供应商，支持筛选、部署、发布与下线 |
| 🔧 控制层 | [接入点管理](接入点管理.md) | 决定"业务怎么调用"和"路由怎么分发" | 公共接入点（平台侧）/ 业务接入点（租户侧）统一管理 |
| 👁 展示层 | [模型广场](模型广场.md) | 面向用户的"模型发现入口" | 精选模型、精选案例、快速指引，展示租户已启用模型 |
| 🔑 凭证层 | [基础设置](基础设置.md) | 决定"谁在调用"和"能调用什么" | API Key 管理、权限绑定、失效联动 |
| 📊 观测层 | [模型观测](模型观测.md) | 让每次调用"看得见、查得到、算得清" | 用量统计（含调用日志明细）+ 多维度统计报表 |
| ⚖ 治理层 | [治理设置](治理设置.md) | 人走了资源清、出事了有人知、没钱了不能用 | 离职清理、通知联系人、调用资格/OM点数包 |

### 工具组模块（3个，仅目录占位）

| 模块 | 子菜单 | 说明 |
|------|--------|------|
| 🧪 [体验中心](体验中心.md) | 大模型 / 小模型 | 零代码快速在线调用模型，验证效果；含历史记录、猜你想问 |
| 🔬 [模型实验室](模型实验室.md) | 小模型训练 / 大模型微调 / 模型评测 / 算法库 | 模型训练、微调、评测与算法管理工具集 |
| 📦 [模型数据](模型数据.md) | 数据集管理 / 数据预处理 / 数据标注 | 数据全生命周期管理，支撑训练与评测数据闭环 |

### 运营管理模块（1个）

| 模块 | 核心职责 | 详情 |
|------|---------|------|
| 🏢 [运营管理](运营管理.md) | 平台级运营配置与全租户用量监控 | 模型计价配置（按量/阶梯）+ 跨租户用量统计与排行 |

## 核心业务流与涉及的模块

### 业务流①：模型上架 → 对用户可见

```
模型管理（注册/上架）
    → 模型广场（精选模型展示，租户已启用模型对用户可见）
```

### 业务流②：业务系统调用模型

```
基础设置（创建API Key，绑定模型/接入点）
    → 接入点管理（通过接入点路由到候选模型）
        → 模型观测（调用日志记录 + 用量统计）
```

### 业务流③：资源关闭生效链

```
资源关闭后 → 基础设置（已有绑定进入"资源关闭失效绑定"状态，调用立即失败）
    → 转发侧同步（从Go token有效权限中移除，清理Redis）
```

### 业务流④：离职人员清理

```
治理设置（识别离职人员创建的API Key）
    → 基础设置（默认禁用 / AI Coding租户自动删除）
        → 转发侧同步（同步禁用/删除状态，清理Redis）
```

> 注意：离职清理**不产生**资源关闭失效绑定，两种联动机制独立。

### 业务流⑤：运营管理 → 计价与跨租户监控

```
运营管理（模型配置 → 按模型ID配置计价规则）
    → 模型观测（用量统计中关联计价信息）
运营管理（租户用量监控 → 跨租户维度统计排行）
```

## 角色权限速查

| 模块 | 普通用户 | 租户管理员 | 平台管理员 |
|------|---------|-----------|-----------|
| [模型广场](模型广场.md) | 可见本租户已启用模型 | 额外可见启用状态 | 可见全平台模型 |
| [体验中心](体验中心.md) | 大模型/小模型在线体验 | 同普通用户 | 同普通用户 |
| [接入点管理](接入点管理.md) | 可见已启用接入点（只读） | 编辑业务接入点 | 编辑公共接入点 |
| [模型管理](模型管理.md) | 不可见或只读 | 不可见或只读 | **完全控制** |
| [模型观测](模型观测.md) | 本人调用/我的用量 | 本租户调用/用量 | 全平台调用/用量 |
| [基础设置](基础设置.md) | 可见本人创建的API Key | 可见本租户API Key | 可见全平台API Key |
| [运营管理](运营管理.md) | 不可见 | 不可见 | **完全控制**（模型配置 + 跨租户监控） |
| [治理设置](治理设置.md) | 不可见 | 本租户治理配置 | 全平台治理配置 |

## 关键术语速查

| 术语 | 定义 | 所属模块 |
|------|------|---------|
| 纳管模型 | 平台正式注册并在模型库中维护的模型 | 模型管理 |
| 公共接入点 | 平台管理员维护的多租户共享调用入口 | 接入点管理 |
| 业务接入点 | 租户管理员维护的单租户专用调用入口 | 接入点管理 |
| 接入点路由意图 | 说明候选模型为什么被放在同一接入点下 | 接入点管理 |
| 资源关闭失效绑定 | 资源关闭后绑定保留但调用立即失效 | 基础设置 |
| 全部资源权限 | API Key对租户当前和未来已启用资源动态生效 | 基础设置 |
| 部分资源权限 | API Key只对显式选择的模型和接入点生效 | 基础设置 |
| 调用明细事件 | 一次模型调用尝试的不可变事实记录 | 模型观测 |
| 调用资格 | 租户是否能调用生产模型的本地判断依据 | 治理设置 |
| OM点数包 | 同时代表调用资格和点数额度的购买包 | 治理设置 |
| 模型配置 | 按模型ID配置的计价规则（按量/阶梯） | 运营管理 |

> 完整术语定义参见：[项目概览/model-ops-knowledge/CONTEXT.md](../项目概览/model-ops-knowledge/CONTEXT.md)
