# 模型广场

| 属性 | 值 |
|------|-----|
| 一级菜单 | 模型广场 |
| 优先级 | P0 |
| 主要用户 | 普通用户、租户管理员、平台管理员 |
| 需求来源 | F1 平台资源池与租户资源管理、F9 模型发布与默认启用策略 |

## 模块定位与职责

模型广场是面向所有用户的**模型发现入口**，展示经过平台资源池和租户资源启用配置过滤后的可用模型。它本身**不维护**模型资产，只负责"展示什么"——数据来源于模型库（模型管理 → 模型库），可见范围受平台资源池和租户资源启用配置双重控制。

## 核心概念

| 概念 | 定义 |
|------|------|
| 模型列表 | 普通用户视角：只展示本租户已启用且可调用的模型 |
| 模型详情 | 展示模型能力、供应商来源摘要、可用状态和模型级调用方式 |
| 实际命中模型 | 公共接入点路由到的具体模型，普通用户在模型广场不一定可见 |

## 角色权限矩阵

| 页面 | 普通用户 | 租户管理员 | 平台管理员 |
|------|---------|-----------|-----------|
| 模型列表 | 可见本租户已启用模型 | 可见本租户已启用模型，可跳转资源管理调整 | 可见全平台模型 |
| 模型详情 | 可见已启用模型的概览/调用方式 | 额外可见本租户资源启用状态 | 额外可见供应商、实例、资源池和默认启用策略 |

## 页面字段清单

### 模型列表

| 字段 | 说明 |
|------|------|
| modelId | 模型标识 |
| providerName | 供应商名称 |
| 能力标签 | 模型能力分类 |
| 启用状态 | 已启用（仅租户管理员可见已关闭模型） |
| 操作 | 查看详情 |

### 模型详情页签

| 页签 | 内容 |
|------|------|
| 概览 | 模型描述、能力参数、上下文窗口、输入输出模式 |
| 供应商来源 | 供应商名称、来源摘要（不暴露供应商密钥和内部配置） |
| 接入方式 | 模型直调 URL、可用公共接入点列表（跳转） |
| 资源状态 | 租户管理员/平台管理员可见：启用开关、资源所属资源池名称 |
| 调用观测 | 跳转调用日志和用量统计 |

## 核心业务规则

1. 普通用户只展示本租户**已启用**且可调用的模型；关闭模型对普通用户不可见
2. 模型广场不作为资源池或启用配置的**事实源**——事实源在资源管理
3. 公共接入点路由命中租户未直接启用模型时，模型广场不展示该模型，也不提供跳转绑定或创建模型级 API Key
4. 关闭模型后，模型广场即时不展示，API Key 不能新增绑定该模型，直接调用 `model=M` 被拒绝
5. 平台管理员可查看全平台模型，包括未分配到任何租户资源池的模型

## 状态流转

模型在模型广场的可见性状态：

```
平台资源池已分配 + 租户已启用 → 可见
平台资源池已分配 + 租户已关闭 → 不可见（租户管理员可见灰色标记）
平台资源池未分配 → 不可见
```

## 与其他模块的上下游关系

| 方向 | 关联模块 | 关系说明 |
|------|---------|---------|
| 上游 | 模型管理 → 模型库 | 模型资产数据的唯一来源 |
| 上游 | 资源管理 | 过滤条件：只有租户已启用资源才在模型广场展示 |
| 上游 | 接入点管理 | 公共接入点的代表模型摘要来自模型库数据 |
| 下游 | 接入凭证 | 模型详情提供"创建模型级 API Key"跳转入口 |
| 下游 | 调用观测 | 模型详情提供"查看调用日志/用量"跳转入口 |

## 关键决策编号及结论

| 决策编号 | 结论 | 影响 |
|---------|------|------|
| D-04c | 不新增普通用户侧"模型开通"菜单 | 模型广场不提供自助开通功能 |
| D-04f | 模型广场只展示租户已启用模型 | 过滤逻辑依赖资源管理数据 |
| D-09 | 调用日志命中模型不扩大模型级授权范围 | 未启用模型不提供绑定/跳转 |
