# F12. 接入凭证列表筛选增强

## 8.12.1 需求说明

API Key 列表（大模型接入凭证）增加**创建人**和**状态**两项筛选，便于租户管理员按人和按状态治理凭证。

## 8.12.2 产品规则

### 创建人筛选

- 筛选控件为租户内用户的搜索下拉控件，按创建人用户 ID 精确过滤；不做姓名自由文本模糊匹配。
- 创建人口径与离职资源清理一致：以 API Key 创建人用户 ID 为准。
- 普通用户列表仍只展示自己创建的凭证，不出现创建人筛选项。

### 状态筛选

- 筛选值枚举与 API Key 状态模型一致，仅包含：生效中、已禁用。
- 支持多选；默认视图不预置状态过滤，避免管理员漏看非启用凭证。
- 普通用户列表保留状态筛选（对自己的凭证有意义），但只出现该用户名下 Key 实际出现过的状态项。

## 8.12.3 边界条件与异常场景

- **创建人为空场景**：API Key 的 `creator_id` 字段为空（极少数历史数据可能缺失）时，该凭证不出现在按创建人筛选的结果中；筛选控件下拉底部提示"含 N 条无创建人记录"，点击可筛选出创建人为空的凭证。
- **租户内用户列表为空**：租户无其他成员时，创建人下拉仅展示当前管理员自己，不影响筛选逻辑。
- **创建人已离职**：已离职用户的 API Key 仍可按其用户 ID 筛选定位到对应的离职待处理凭证。
- **状态枚举扩容**：后续新增 API Key 状态时，筛选枚举自动同步，无需手动维护花名册。
- **并发筛选**：多用户同时在不同终端执行筛选操作，互不影响，结果各自独立。

## 8.12.4 字段定义

### 页面展示字段 — 筛选控件

| 字段 | 说明 | 是否本期新增 |
|------|------|:----------:|
| 创建人筛选 | 租户内用户搜索下拉，选中后按 `creator_id` 精确过滤 | 是 |
| 状态筛选 | 多选下拉，枚举值与 API Key 状态模型一致 | 是 |

### 页面展示字段 — 创建人下拉选项

| 字段 | 说明 | 是否本期新增 |
|------|------|:----------:|
| 用户 ID | 创建人的唯一标识，支持搜索 | 否（已有） |
| 姓名 | 创建人显示姓名，支持搜索 | 否（已有） |

### 页面展示字段 — 状态筛选枚举

| 枚举值 | 说明 | 是否本期新增 |
|--------|------|:----------:|
| 生效中 | API Key 当前可用 | 否（已有） |
| 已禁用 | 手动禁用 | 否（已有） |

### API 字段

| 字段 | 说明 | 是否本期新增 |
|------|------|:----------:|
| creator_id | 创建人用户 ID，精确过滤参数 | 否（已有字段，新增筛选参数） |
| status | API Key 状态枚举，多选过滤参数 | 否（已有字段，新增筛选参数） |

### 数据库字段

| 字段 | 说明 | 是否本期新增 | 存储 |
|------|------|:----------:|------|
| creator_id | API Key 创建人用户 ID | 否（已有） | API Key 表 |
| status | API Key 状态枚举 | 否（已有） | API Key 表 |

## 8.12.5 页面交互

### 创建人筛选控件

- 控件类型：可搜索下拉选择器。
- 触发方式：点击筛选区"创建人"标签激活下拉。
- 搜索行为：输入姓名或用户 ID 后，下拉实时模糊匹配租户内用户列表（按姓名 / 用户 ID 搜索定位），选中后按 `creator_id` 精确过滤。
- 选中展示：选中后在筛选区以 Tag 形式展示用户姓名，支持清除。
- 普通用户列表：不展示该筛选项。

### 状态筛选控件

- 控件类型：多选下拉选择器。
- 触发方式：点击筛选区"状态"标签激活下拉。
- 选项展示：枚举值按固定顺序排列，租户管理员展示全部枚举；普通用户仅展示其名下 Key 实际出现过的状态项。
- 默认行为：不预置任何选中值，展示全部状态凭证。
- 选中展示：选中多个状态后在筛选区以 Tag 形式展示，支持逐个清除或一键清空。
- 普通用户列表：展示该筛选项，但选项集合动态收窄至该用户名下实际存在的状态。

### 创建人为空提示

- 当租户内存在 `creator_id` 为空的 API Key 时，创建人下拉底部展示"含 N 条无创建人记录"提示项，点击后筛选出创建人为空的凭证列表。

## 8.12.6 状态流转

本功能不涉及状态流转变更，筛选仅基于 API Key 现有状态模型读取。状态模型定义见 F4（转发侧同步）与 F7（离职资源清理）。

## 8.12.7 验收标准

### 一、创建人筛选（AC-12.1 ~ AC-12.2）

- [ ] **AC-12.1** Given 租户管理员在凭证列表页；When 选择创建人筛选并选中某用户；Then 列表仅展示该用户 ID 创建的凭证，结果精确匹配。
- [ ] **AC-12.2** Given 租户管理员点击创建人筛选下拉；When 输入某用户的姓名或用户 ID；Then 下拉实时返回匹配的用户选项供选择。

### 二、状态筛选（AC-12.3 ~ AC-12.5）

- [ ] **AC-12.3** Given 租户管理员在凭证列表页；When 选择状态筛选并选中多个状态值；Then 列表展示符合任一选中状态的凭证。
- [ ] **AC-12.4** Given 状态筛选枚举值；When 与 API Key 状态模型对比；Then 枚举值完全对应，无遗漏无多余。
- [ ] **AC-12.5** Given 普通用户在凭证列表页；When 展开状态筛选下拉；Then 仅展示该用户名下 Key 实际出现过的状态项。

### 三、权限控制（AC-12.6 ~ AC-12.8）

- [ ] **AC-12.6** Given 普通用户在凭证列表页；When 查看筛选区域；Then 不展示创建人筛选项。
- [ ] **AC-12.7** Given 租户内存在无创建人记录的 API Key；When 管理员展开创建人下拉；Then 下拉底部展示"含 N 条无创建人记录"提示项，点击可筛选出对应凭证。
- [ ] **AC-12.8** Given 已离职用户的 API Key 仍在系统中；When 管理员在创建人下拉中搜索或浏览；Then 该用户正常展示，不额外标注离职状态。

## 8.12.8 补充约束

- 创建人下拉的用户列表来源于 IAM 系统同步数据，搜索延迟不得超过 500ms；若 IAM 接口超时则下拉提示"用户列表加载失败，请稍后重试"，不阻塞其他筛选操作。
- 状态筛选枚举由后端 API 动态返回，前端不做硬编码；后续新增状态时前端自动适配。
- 创建人与状态两项筛选可组合使用，组合逻辑为 AND。
- 每次进入页面默认为全量无筛选视图。
