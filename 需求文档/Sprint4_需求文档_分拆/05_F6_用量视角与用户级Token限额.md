# F6. 用量视角与用户级 Token 限额

## 8.6.1 需求说明

- **功能目标**：为不同角色提供差异化用量视角，并对启用治理租户下的用户执行月度 Token 用量限额，快速控制 AI Coding 等高消耗场景中的异常个人成本。
- **触发条件**：用户访问用量页面时按角色展示对应视角；启用租户下的用户每次发起模型调用前置校验限额。
- **前置依赖**：ES 调用日志查询索引可用（F5）；Redis 可用于限额计数；限额启用租户名单和用户级限额配置已在后台维护。
- **涉及角色**：普通用户（查看用量）、租户管理员（查看本租户用量 / 一键删除离职 API Key）、平台管理员（查看全平台用量 / 维护限额配置）。

## 8.6.2 产品规则

### 用量视角

- PR-6.1：普通用户只可查看”我的用量”，无可切换视角。
- PR-6.2：租户管理员默认查看”本租户用量”，可切换到”我的用量”。
- PR-6.3：平台管理员默认查看”全平台用量”，可切换到”指定租户”或”我的用量”。
- PR-6.4：页面必须明确展示当前视角，避免用户误判统计范围。

### 统计维度

- PR-6.5：用量统计至少支持：时间范围、租户、用户、API Key、接入点、实际命中模型、供应商来源、调用状态、Token 用量、失败率。
- PR-6.6：公共接入点调用命中租户未直接启用模型时，用量统计仍可按实际命中模型筛选和聚合，但不提供跳转到未启用模型详情或绑定操作。
- PR-6.6a：公共接入点调用被 AI 安全护栏拦截时，调用状态记为 `FAILED`，计入总调用数、失败调用数和失败率；判官模型超时后 fail-open 且业务调用成功时，不因判官超时记为失败。

### 用户级月度 Token 限额

- PR-6.7：限额主体是用户，不是单把 API Key；用户创建多把 API Key 时，所有 API Key 产生的 Token 用量合并到该用户的月度限额中。
- PR-6.8：归属键产品语义为 API Key 创建人；实现层以 ES 调用日志的 `create_id` 字段为归属键。`api_key_owner_id` 字段填充率不足 1%、语义与 `create_id` 重复，评估随 F5 索引拆分废弃或与 `create_id` 统一。
- PR-6.9：只有进入「平台限额启用租户名单」的租户启用用户级月度 Token 限额治理；名单外的租户不受限额约束。首批启用租户：`CL0100` / `CL0200` / `CL0300` / `CL0400` / `CL0500` / `CL0600` / `CL0800` / `CL0900` / `CL1100`（9 家）；名单由平台管理员通过后台配置维护，可增删。
- PR-6.10：默认额度 `500,000,000 Tokens / 自然月`（500M / 月），可通过后台配置调整。初始值以 2026-06 上月候选启用租户 P92～P95 位为锚，投产后按运行数据回顾调整。
- PR-6.11：额度生效顺序——启用租户内被后台单独配额度的用户按其单独额度阻断；未被单独配额度的用户走平台默认额度阻断（本期改动：原”未配置用户默认不限额”改为”走默认额度”）；额度为 0 表示显式禁止该用户继续产生受限范围内的模型调用成本；删除某用户单独配置回到走默认额度。
- PR-6.12：只累加 `status=SUCCESS` 的调用；`RATE_LIMITED` 与 `FAILED` 均不计入。Token 口径为 `tokens_total`（含缓存读写）。私有模型的限额同样计入用户额度内，不做豁免。
- PR-6.13：短解不新增完整配置页面，平台管理员通过后台配置表或运维方式维护启用租户名单和用户级限额配置。
- PR-6.14：按自然月统计；进入新自然月后已用 Token 归零，额度配置继续沿用。
- PR-6.15：达到限额时调用前置校验拒绝继续调用，返回不可重试失败，不触发接入点路由切换。
- PR-6.16：模型输出 Token 只能在调用完成后确认，允许最后一次已放行调用使实际用量轻微超过限额。

### 限额数据与一致性

- PR-6.17：调用前置校验不扫描调用日志或查询调用明细来判断是否超额；短解使用 Redis 月度用户 Token 计数作为主链路快速判断口径。
- PR-6.18：Redis 不是限额用量的唯一事实源；系统需要持久化月度用户 Token 账本，用于恢复、对账和后台运维查询。
- PR-6.19：月度用户 Token 账本按自然月、租户和用户累计，不按 API Key 分配额度；API Key 维度仅用于调用明细审计和排查。
- PR-6.20：限额用量更新只累加 `status=SUCCESS` 的调用，并必须以调用明细事件 ID 或 `request_id` 作为幂等依据，避免重复消费导致重复累计。
- PR-6.21：当 Redis 月度用户 Token 计数与月度用户 Token 账本不一致时，以持久化账本为准重建或修正 Redis。
- PR-6.22：限额配置必须保存为可审计的持久化记录，包含配置主体、额度、启停状态、操作人和操作时间。
- PR-6.23：限额配置新增、停用或额度调整后，应在下一次调用前置校验中生效，不能只依赖缓存自然过期。
- PR-6.24：同一模型调用在同一时间只能命中一套成本限额策略，避免短解和长解重复阻断。

## 8.6.3 边界条件与异常场景

- **Redis 不可用**：当 Redis 不可用导致无法完成限额校验时，短解默认放行并记录平台侧异常，恢复后通过月度用户 Token 账本修正计数。
- **月末归零**：进入新自然月后已用 Token 归零，需保证月末最后时刻的并发调用在归零操作期间不丢失也不重复累计；归零操作应在月初首次请求时延迟执行，避免跨月边界竞态。
- **配置生效窗口**：限额配置新增、停用或额度调整后，需在下一次调用前置校验中生效；若配置更新与并发调用同时发生，允许短暂窗口内按旧配置放行，但不允许配置永久滞后。

## 8.6.5 页面交互

- **操作入口**：左侧导航栏”用量统计”菜单；用量页面右上角视角切换下拉。
- **交互流程**：
  1. 用户进入用量页面，默认按角色展示对应视角数据。
  2. 租户管理员 / 平台管理员可通过右上角下拉切换视角，切换后页面标题和面包屑同步更新。
  3. 启用租户下的用户在用量页面可见”我的限额”卡片，展示已用额度 / 总额度和进度条。
  4. 当用户月度 Token 接近限额（达 90%）时，”我的限额”卡片展示黄色预警提示。
  5. 用户月度 Token 达到限额后，调用返回超额错误，前端弹出超额提示。
- **二次确认**：视角切换无需二次确认；限额配置变更由平台管理员在后台操作，不涉及前端二次确认。
- **提示 / 错误文案**：
  - 超额预警（90%）：”您本月模型调用额度已使用超过 90%，请关注用量”
  - 超额拦截：”当前用户本月模型调用额度已用尽，请等待下月恢复或联系平台管理员调整额度”
  - 视角标识：页面顶部面包屑展示”用量统计 > {视角名称}”

## 8.6.6 状态流转

本功能中的”限额状态”无独立生命周期，用户月度限额随自然月周期性归零重启，不适用 Mermaid 状态图。

## 8.6.7 验收标准

### 一、用量视角（AC-6.1 ~ AC-6.4）

- [ ] **AC-6.1** Given 普通用户登录；When 进入用量页面；Then 只能查看”我的用量”，无可切换视角。
- [ ] **AC-6.2** Given 租户管理员登录；When 进入用量页面；Then 默认展示”本租户用量”，可切换到”我的用量”。
- [ ] **AC-6.3** Given 平台管理员登录；When 进入用量页面；Then 默认展示”全平台用量”，可切换到”指定租户”或”我的用量”。
- [ ] **AC-6.4** Given 用户通过公共接入点命中未启用模型；When 查看用量统计；Then 可按实际命中模型筛选但不提供详情跳转。

### 二、用户级 Token 限额（AC-6.5 ~ AC-6.8）

- [ ] **AC-6.5** Given 启用租户名单内且已单独配置额度的用户；When 月度 Token 达到额度；Then 后续调用被前置拒绝。
- [ ] **AC-6.6** Given 启用租户内未被单独配置额度的用户；When 月度 Token 达到默认额度（500M/月，可配）；Then 后续调用被前置拒绝。
- [ ] **AC-6.7** Given 同一用户创建多把 API Key；When 产生 Token 用量；Then 共享同一个用户级月度 Token 限额。
- [ ] **AC-6.8** Given 不在启用租户名单内的租户；When 其用户发起调用；Then 不受限额约束但用量进入统计。

### 三、限额计数与周期（AC-6.9 ~ AC-6.13）

- [ ] **AC-6.9** Given 只有 `status=SUCCESS` 的调用；When 计入月度用户 Token 账本；Then `RATE_LIMITED` 和 `FAILED` 不计入。
- [ ] **AC-6.10** Given 限额用量累计；When 以 `request_id` 幂等判断；Then 重复消费不会重复累计。
- [ ] **AC-6.11** Given Redis 限额计数不可用；When 短解默认放行；Then 记录平台侧异常，恢复后通过账本修正计数。
- [ ] **AC-6.12** Given 进入新自然月；When 用户首次发起调用；Then 已用 Token 归零，额度配置继续沿用。
- [ ] **AC-6.13** Given 限额配置新增、停用或额度调整；When 下一次调用前置校验；Then 按新配置生效。

## 8.6.8 补充约束

- 限额短解仅为本期止血方案，不作为后续长效架构的基础约束；长解方案演进时短解可整体替换。
- 启用租户名单变更需记录操作审计日志。
- 月度用户 Token 账本的数据保留期限建议与查询索引一致（12 个月），以便对账回溯。
