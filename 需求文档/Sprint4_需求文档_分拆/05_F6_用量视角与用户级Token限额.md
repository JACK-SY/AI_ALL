# F6. 用量视角与用户级 Token 用量预警

> **2026-07-21 口径调整**：本期交付由「网关侧实时阻断式月度限额」调整为 **纯 Java 侧定时统计 + 超额邮件预警**——**不拦截 / 不限流任何调用、不改动网关链路、不做调用前置校验**。阻断式成本硬控移出本期、保留为后续成本治理演进方向。

## 8.6.1 需求说明

- **功能目标**：为不同角色提供差异化用量视角，并对治理租户名单内的用户按自然日/自然月统计 Token 用量，超过阈值时发送邮件预警，实现成本可观测与人工干预触达。
- **触发条件**：用户访问用量页面时按角色展示对应视角；定时任务每小时巡检治理租户名单内用户的用量，超过阈值时发送预警邮件。
- **前置依赖**：ES 调用日志查询索引可用（F5）；治理租户名单和预警规则已在后台维护；EMC 邮件服务可用。
- **涉及角色**：普通用户（查看用量）、租户管理员（查看本租户用量）、平台管理员（查看全平台用量 / 维护治理名单和预警规则）、超额用户（接收预警邮件）。

## 8.6.2 产品规则

### 用量视角

- PR-6.1：普通用户只可查看”我的用量”，无可切换视角。
- PR-6.2：租户管理员默认查看”本租户用量”，可切换到”我的用量”。
- PR-6.3：平台管理员默认查看”全平台用量”，可切换到”指定租户”或”我的用量”。
- PR-6.4：页面必须明确展示当前视角，避免用户误判统计范围。

### 统计维度

- PR-6.5：用量统计至少支持：时间范围、租户、用户、API Key、接入点、实际命中模型、供应商来源、调用状态、Token 用量、失败率。
- PR-6.6：公共接入点调用命中租户未直接启用模型时，用量统计仍可按实际命中模型筛选和聚合，但不提供跳转到未启用模型详情或绑定操作。
- PR-6.6a：公共接入点调用被 AI 安全护栏拦截时，调用状态记为 `FAILED`，计入总调用数、失败调用数和失败率；判官模型超时后 fail-open 且业务调用成功时，不因判官超时记为失败。

### 用户级 Token 用量预警

- PR-6.7：预警主体是用户，不是单把 API Key；用户创建多把 API Key 时，所有 API Key 产生的 Token 用量合并到该用户的预警统计中。
- PR-6.8：归属键产品语义为 API Key 创建人；实现层以 ES 调用日志的 `create_id` 字段为归属键，同时作为预警邮件收件人。`api_key_owner_id` 字段填充率不足 1%、语义与 `create_id` 重复，评估随 F5 索引拆分废弃或与 `create_id` 统一。
- PR-6.9：只有进入「治理租户名单」的租户纳入用量预警治理；名单外的租户不巡检、不发送预警。首批治理租户：`CL0100` / `CL0200` / `CL0300` / `CL0400` / `CL0500` / `CL0600` / `CL0800` / `CL0900` / `CL1100`（9 家）；名单由平台管理员通过后台配置维护，可增删。
- PR-6.10：预警规则包含生效租户范围（= 治理名单）、modelId 名单（巡检仅统计名单内 modelId 的调用）、日阈值（默认 2500 万 Tokens）、月阈值（默认 2.5 亿 Tokens）。规则落业务库配置表，可后台调整。
- PR-6.11：只累加 `status=SUCCESS` 且 `model_id ∈ 规则 modelId 名单` 的调用；`RATE_LIMITED` 与 `FAILED` 均不计入。Token 口径为 `tokens_total`（含缓存读写与输出）。
- PR-6.12：预警邮件发送规则——日线按「人×自然日」、月线按「人×自然月」各最多一封；同轮日/月双超合并为一封（alert_type=BOTH）。
- PR-6.13：预警邮件收件人为超额用户本人（`create_id`），抄送平台管理员（固定名单，落配置文件）。
- PR-6.14：预警触发方式——Spring 定时任务（默认每小时一次）；另提供手动触发接口用于联调/补发。
- PR-6.15：预警发送失败处理——发送异常或响应无 Success 视为未发出，记 FAILED + fail_reason，不兑现去重，下一轮巡检自动重试。
- PR-6.16：不同租户下同一用户用量独立统计，互不影响。

### 预警数据与一致性

- PR-6.17：用量统计取自 ES 调用日志（既有事实源），不依赖 Redis；预警发送记录落业务库 `usage_alert_notify_record`，作为去重依据和审计台账。
- PR-6.18：预警发送记录包含 alert_type（DAILY/MONTHLY/BOTH）、用量快照、阈值快照、收件人、抄送、发送状态、失败原因。
- PR-6.19：周期归零由统计时间窗与记录维度驱动——日线随自然日切日、月线随自然月切月各自归零，无需清理任务。
- PR-6.20：巡检无状态、可重复执行；每次发送前重查对应维度是否已成功，重复巡检不会重复发送。
- PR-6.21：预警规则配置必须保存为可审计的持久化记录，包含规则主体、modelId 名单、阈值、操作人和操作时间。

## 8.6.3 边界条件与异常场景

- **EMC 邮件服务不可用**：当 EMC 邮件服务不可用导致预警邮件发送失败时，记 FAILED + fail_reason，视为未发出，下一轮巡检自动重试；全程不影响模型调用链路。
- **周期归零**：日线随自然日切日、月线随自然月切月各自归零，由统计时间窗与记录维度驱动，无需清理任务。
- **配置生效窗口**：预警规则配置变更后，在下一次巡检任务执行时生效；修改不影响正在进行的巡检。
- **同轮双超合并**：同一用户在同一轮巡检中日用量和月用量同时超过阈值时，合并为一封邮件发送（alert_type=BOTH），同时兑现日/月维度的去重。

## 8.6.5 页面交互

- **操作入口**：左侧导航栏”用量统计”菜单；用量页面右上角视角切换下拉。
- **交互流程**：
  1. 用户进入用量页面，默认按角色展示对应视角数据。
  2. 租户管理员 / 平台管理员可通过右上角下拉切换视角，切换后页面标题和面包屑同步更新。
- **二次确认**：视角切换无需二次确认；预警规则和治理名单配置由平台管理员在后台操作，不涉及前端二次确认。
- **提示 / 错误文案**：
  - 视角标识：页面顶部面包屑展示”用量统计 > {视角名称}”

## 8.6.6 状态流转

本功能中的”限额状态”无独立生命周期，用户月度限额随自然月周期性归零重启，不适用 Mermaid 状态图。

## 8.6.7 验收标准

### 一、用量视角（AC-6.1 ~ AC-6.4）

- [ ] **AC-6.1** Given 普通用户登录；When 进入用量页面；Then 只能查看”我的用量”，无可切换视角。
- [ ] **AC-6.2** Given 租户管理员登录；When 进入用量页面；Then 默认展示”本租户用量”，可切换到”我的用量”。
- [ ] **AC-6.3** Given 平台管理员登录；When 进入用量页面；Then 默认展示”全平台用量”，可切换到”指定租户”或”我的用量”。
- [ ] **AC-6.4** Given 用户通过公共接入点命中未启用模型；When 查看用量统计；Then 可按实际命中模型筛选但不提供详情跳转。

### 二、用户级 Token 用量预警（AC-6.5 ~ AC-6.10）

- [ ] **AC-6.5** Given 治理租户名单内用户日用量超过阈值；When 巡检任务执行；Then 发送日用量预警邮件给用户本人，抄送平台管理员。
- [ ] **AC-6.6** Given 治理租户名单内用户月用量超过阈值；When 巡检任务执行；Then 发送月用量预警邮件给用户本人，抄送平台管理员。
- [ ] **AC-6.7** Given 治理租户名单内用户日用量和月用量同时超过阈值；When 巡检任务执行；Then 合并为一封邮件发送（alert_type=BOTH），不分别发送。
- [ ] **AC-6.8** Given 不在治理租户名单内的租户用户；When 用量超过阈值；Then 不发送预警邮件。
- [ ] **AC-6.9** Given 同一用户创建多把 API Key；When 产生 Token 用量；Then 合并统计到该用户的预警统计中。
- [ ] **AC-6.10** Given 同一用户不同租户下的用量；When 统计预警；Then 不同租户用量独立统计，互不影响。

### 三、预警计数与周期（AC-6.11 ~ AC-6.15）

- [ ] **AC-6.11** Given 只有 `status=SUCCESS` 且 `model_id ∈ modelId 名单` 的调用；When 计入预警统计；Then `RATE_LIMITED` 和 `FAILED` 不计入。
- [ ] **AC-6.12** Given 同一用户当日已成功发送日预警；When 再次触发日用量超过阈值；Then 不再重复发送当日预警邮件。
- [ ] **AC-6.13** Given EMC 邮件服务不可用；When 预警邮件发送失败；Then 记 FAILED + fail_reason，下一轮巡检自动重试。
- [ ] **AC-6.14** Given 进入新自然日/自然月；When 巡检任务执行；Then 日/月维度预警重新开始计数。
- [ ] **AC-6.15** Given 预警规则配置变更；When 下一次巡检任务执行；Then 按新配置生效。

## 8.6.8 补充约束

- 本期用量预警仅作为成本可观测与人工干预触达手段，不拦截/不限流任何调用；阻断式成本硬控保留为后续演进方向。
- 治理租户名单变更需记录操作审计日志。
- 预警发送记录的数据保留期限建议与查询索引一致（12 个月），以便对账回溯。
- 预警邮件模板由平台统一维护，本期不支持用户自定义。
