# F8. AI 安全护栏接入点策略

> 状态：本期。2026-07-15 产品确认：三个防护维度全部平台自研，不接入第三方检测服务——内容合规/提示词攻击由自托管安全判官模型承担，敏感内容由网关本地规则引擎承担；标签体系沿用为平台自有体系。

## 8.8.1 需求说明

### 功能目标

AI 安全护栏作为接入点治理能力提供，对经过接入点的模型调用执行输入/输出安全检测，支持观察与拦截两种运行模式，降低违规内容、敏感信息泄露与提示词攻击风险。

### 触发条件

调用目标为 `ep-*`（经由接入点）时，网关解析到接入点后加载该接入点绑定的策略执行检测；不经接入点的模型直调（模型级能力授权路径）本期不经过 AI 安全护栏。

### 前置依赖

- 接入点已创建（策略随接入点生命周期创建，默认未启用）；
- 自托管安全判官模型（内容合规/提示词攻击维度）已就绪，网关本地规则引擎（敏感内容维度）已实现；
- 网关已集成护栏策略加载与检测调用逻辑（对齐技术边界说明调用链路第 5 步「AI 安全护栏策略通过」）。

### 涉及角色

- 平台管理员：维护公共接入点策略；
- 租户管理员：随接入点编辑维护业务接入点策略。

### 映射产品目标

v1 不默认启用，避免租户无感知被拦截；平台不自动套用任何策略，只提供推荐配置（D-29）。满足模型调用安全合规要求，为接入点提供可配置、可观测的安全防护层，实现租户级差化安全策略。

## 8.8.2 产品规则

### 绑定关系

- 一个接入点绑定一条接入点策略；策略是该接入点自己的护栏配置（检测服务启停、防护维度开关、标签配置、运行模式），不是跨接入点共享的独立实体，不同接入点的策略互不影响。
- 推荐配置是平台维护的模板：套用后模板内容落成该接入点自己的策略，后续各自修改互不联动（对齐 D-29：平台不自动套用任何策略，只提供推荐配置）。
- 公共接入点与业务接入点均支持绑定：公共接入点策略由平台管理员维护，业务接入点策略由租户管理员随接入点编辑维护。
- 策略随接入点生命周期：随接入点创建（默认未启用）、随接入点删除清理；接入点关闭期间策略配置保留。
- 创建公共接入点时可在接入点编辑/创建页勾选「套用 AI 安全护栏推荐配置」，默认不勾选，勾选后以观察模式生效。
- 护栏配置入口保持在接入点配置内，本期不提供独立一级菜单（对齐非目标）。

### 调用链路结合

（对齐技术边界说明调用链路第 5 步「AI 安全护栏策略通过」）

- 调用目标为 `ep-*` 时，网关解析到接入点后加载该接入点绑定的策略执行检测：输入检测在转发上游前执行，拦截模式命中高风险时直接返回安全响应，不再消耗上游模型调用；输出检测在返回调用方前执行。
- 未启用护栏（或未绑定策略）的接入点跳过检测，不产生护栏附加延迟。
- 不经接入点的模型直调（模型级能力授权路径）本期不经过 AI 安全护栏。

### 状态外显

- 公共接入点详情展示护栏状态：未启用 / 观察模式 / 拦截模式（对齐 8.2.5 公共接入点详情）。

### 策略与运行规则

- 新建接入点默认不启用 AI 安全护栏；未启用护栏的接入点调用跳过检测，无护栏附加延迟。
- 页面引导用户套用推荐配置；推荐配置 = 开启输入 / 输出两个检测服务 + 三个防护维度 + 各维度默认标签集与默认风险分段。
- 首次套用推荐配置后，默认进入观察模式；用户可从观察模式切换到拦截模式。
- 观察模式只记录风险结果，不阻断模型调用，也不改动请求 / 响应内容（敏感内容命中同样只记录，不脱敏）。
- 拦截模式下：内容合规、提示词攻击命中高风险时阻断调用并返回安全响应，中 / 低风险只记录；敏感内容命中已开启标签时执行脱敏后继续调用。
- 本期护栏检测能力按四层组织：检测服务（决定检测方向与模态）→ 防护维度（决定风险类型分组）→ 细分检测类别（维度配置页左侧导航）→ 标签（细分场景，携带风险分段、检测范围与处置方式）。接入点策略就是对这四层结构的一份具体配置，外加运行模式（观察 / 拦截）。
- 一条接入点策略同时管理输入、输出两个检测服务，两个方向可独立启停。
- 网关调用检测接口时通过 `service` 参数指定检测服务。
- 每个防护维度在检测服务内独立开关，开启后经「配置管理」进入维度配置页。
- 拦截模式下敏感内容处置方式本期固定为脱敏：命中已开启标签时，把命中字段替换为脱敏文本后继续调用链路（输入方向脱敏后转发，输出方向脱敏后返回），不整条拦截；按等级拦截（如仅拦截 S3）列入后续评估。
- 全部检测项关闭，或检测计算后未发现异常时，检测结果返回 `nonLabel`。
- AI 安全护栏拦截属于不可重试失败，不触发接入点候选切换（对齐 8.2.6）。
- 拦截返回的安全响应使用平台错误码，纳入 F16 网关错误码 i18n 范围（文案清单挂 T17）。
- 每次检测结果写入调用明细事件的安全护栏结果字段，至少包含：检测方向（输入 / 输出）、防护维度、细分检测类别、标签值、风险分（如适用）、风险等级或标签等级、处置动作（记录 / 脱敏 / 拦截）。
- 未命中任何标签时记录 `nonLabel`，表示本次检测无异常。
- 平台管理员可通过 `request_id` 在调用日志查询护栏结果（对齐场景 7）。
- 图片、文件与多模态检测服务（AIGC 输入 / 输出图片安全检测、文件实时检测、图文混审等）不在本期范围。
- 原口径中单列的「敏感信息泄露」不再作为独立检测项：个人 / 企业敏感信息识别由敏感内容维度覆盖，泄露与投毒类攻击由提示词攻击维度的「数据泄露与投毒」类别覆盖。
- 本期不引入：恶意 URL 维度、模型幻觉维度（检测服务侧公测中）、自定义检测 Agent、知名 IP 检测 Agent、自定义词库。

### 检测结果记录规则

- 安全护栏结果作为调用明细事件的一个字段被记录，是用量统计和调用日志查询的事实来源。
- 无论调用是否被拦截，每次检测结果均写入调用明细事件，包括：
  - 未启用护栏的接入点调用：不产生护栏结果字段；
  - 启用护栏但未命中任何标签：记录 `nonLabel`；
  - 观察模式命中风险：记录风险结果，处置动作标记为「记录」；
  - 拦截模式命中高风险：记录风险结果，处置动作标记为「拦截」；
  - 拦截模式命中中/低风险：记录风险结果，处置动作标记为「记录」；
  - 敏感内容命中已开启标签：记录风险结果，处置动作标记为「脱敏」。

## 8.8.3 边界条件与异常场景

### 输入边界

- 活跃接入点的护栏策略配置项（风险分段、标签开关等）无字符数长度额外限制，遵循检测服务侧标签值与范围定义。
- 标签等级限于 S1\~S3 配置（S0 不参与检测）。

### 并发竞态

- 同一接入点策略被多角色同时编辑时，采用后写入覆盖策略，不做行级锁；配置变更写审计日志。

### 降级容灾

- **检测服务超时**：网关调用检测服务超时时，按观察模式行为放行请求并记录超时事件，不阻断调用。
- **检测服务不可用**：检测服务整体不可用（如服务宕机、网络中断）时，降级为观察模式放行所有请求，并记录降级事件；降级期间所有策略命中均仅记录，不执行拦截或脱敏。
- 检测服务恢复后自动恢复正常策略行为，无需人工干预。

### 流式响应截断边界

- 流式响应输出检测命中拦截条件时，已发送至调用方的 SSE 片段不可撤回，截断在下一个检测窗口执行；调用方收到的响应为部分内容 + 流结束标记 + 安全响应尾部提示（对齐 T18）

### 配置生效窗口期

- 配置修改对生产环境生效通常需要 2\~5 分钟；此窗口期内，新旧策略可能共存：
  - 请求 A 使用旧策略（编辑前配置），请求 B 使用新策略（编辑后配置），均属正常行为；
  - 编辑操作需在配置页明示生效窗口并提示谨慎操作。

### 存量兼容

- 存量接入点（Sprint 4 之前创建）策略默认为「未启用」，需手动开启并配置；不存在自动迁移或强制激活。

### 敏感内容维度标签字段

敏感等级沿用 S0\~S3 分级：

| 等级 | 定义                     | 示例            |
| -- | ---------------------- | ------------- |
| S0 | 不敏感数据，公开在绝大多数情况下不会造成危害 | 省份、城市、商品名称    |
| S1 | 一般敏感数据，不适合公开，泄露危害程度较低  | 固定电话、邮箱       |
| S2 | 关键敏感数据，泄露会带来经济损失或其他危害  | 各类身份证件信息、账号密码 |
| S3 | 核心机密数据，任何情况下不应泄露       | —             |

| 1880 | 法国社会保险号码（INSEE）  | S2   | 开启   | 脱敏   |
| 1879 | 驾驶证号（日本）         | S2   | 开启   | 脱敏   |

配置能力与处置语义：支持一键「启用推荐配置方案」，也支持按标签编辑检测状态。敏感内容维度不使用风险分，命中即按标签等级与处置方式处理。处置方式本期固定为脱敏：拦截模式下命中已开启标签时，把命中字段替换为脱敏文本后继续调用链路（输入方向脱敏后转发，输出方向脱敏后返回），不整条拦截；按等级拦截（如仅拦截 S3）列入后续评估。

### 提示词攻击维度标签字段

| 字段       | 说明                                                            | 是否本期新增 |
| -------- | ------------------------------------------------------------- | :----: |
| 标签值      | 英文标签标识，命中时随检测结果返回                                             |    是   |
| 含义       | 攻击类型中文含义                                                      |    是   |
| 低/中/高风险分 | 检测分值到风险等级的映射区间；v1 只读（三级严重度直接映射：Unsafe=高、Controversial=中） |    是   |
| 标签描述     | 标签的详细说明                                                       |    是   |
| 检测状态     | 已开启 / 已关闭                                                     |    是   |

**「提示词注入」类别标签**（默认全部开启）：

| 标签值                          | 含义      | 标签描述                                                                  |
| ---------------------------- | ------- | --------------------------------------------------------------------- |
| `Direct Prompt Injection`    | 直接提示词注入 | 在输入中直接插入恶意指令，覆盖模型原有规则，诱导生成特定有害内容；支持对模拟对话注入、对立响应注入、道德绑架注入、语种注入等攻击方式的检测 |
| `Indirect Prompt Injection`  | 间接提示词注入 | 通过第三方数据源（在线网页、外部文档等）间接引入恶意指令；支持的攻击方式检测同上                              |
| `Recursive Prompt Injection` | 递归提示词注入 | 提示词对应的输出内容本身包含注入指令，可连锁攻击其它接受该输出作为输入的 LLM 系统                           |
| `Prompt Leaking`             | 提示词泄漏   | 诱导模型泄漏预设的系统提示词                                                        |

其余 5 类（越狱、算力消耗、恶意操作、身份与会话异常、数据泄露与投毒）标签结构相同，标签清单以检测服务标签明细为准。

### 调用明细事件安全护栏结果字段

安全护栏结果作为调用明细事件的一个字段被记录，是用量统计和调用日志查询的事实来源。

| 字段     | 说明                                      | 是否本期新增 |
| ------ | --------------------------------------- | :----: |
| 检测方向   | 输入 / 输出                                 |    是   |
| 防护维度   | 内容合规 / 敏感内容 / 提示词攻击                       |    是   |
| 细分检测类别 | 如「色情内容检测」「个人身份信息」「提示词注入」等                |    是   |
| 标签值    | 英文标签标识（如 `pornographic_adult_activity`）或数字标签 ID（如 1927） |    是   |
| 风险分    | 检测分值（内容合规、提示词攻击维度适用，敏感内容维度无此字段）             |    是   |
| 风险等级   | 低 / 中 / 高（内容合规、提示词攻击）或 S1\~S3（敏感内容）            |    是   |
| 处置动作   | 记录 / 脱敏 / 拦截                              |    是   |
| nonLabel | 未命中任何标签时返回，值为 true 表示本次检测无异常              |    是   |

## 8.8.5 页面交互

### 操作入口

- 护栏配置入口在接入点详情 / 编辑页的「AI 安全护栏」配置区块；
- 公共接入点创建时可在接入点编辑/创建页勾选「套用 AI 安全护栏推荐配置」，默认不勾选。

### 交互流程

- 页面层级：护栏区块 → 检测服务（输入 / 输出）→ 防护维度卡片（开关 + 「配置管理」入口）→ 维度配置页（左侧细分检测类别导航 + 右侧标签配置表）。
- 标签配置表默认只读，点「编辑」进入编辑态。
- 内容合规、提示词攻击维度风险分段 v1 只读，不提供「恢复默认分值」。
- 敏感内容维度提供「启用推荐配置方案」。
- 配置修改对生产环境生效通常需要 2\~5 分钟，配置页需明示生效窗口并提示谨慎操作。
- 公共接入点由平台管理员维护，业务接入点由租户管理员随接入点编辑维护；策略与标签配置变更写审计日志。

### 二次确认

- 从观察模式切换到拦截模式需二次确认，提示文案：「切换为拦截模式后，命中高风险的调用将被阻断，敏感内容将被脱敏处理。确认切换？」\[待确认:T-07]
- 关闭整个护栏（从启用状态切换到未启用）需二次确认。\[待确认:T-08]

### 提示文案

- 配置生效提示：「配置修改约需 2\~5 分钟生效，请谨慎操作。」
- 推荐配置套用提示：「推荐配置将以观察模式生效，包含输入/输出检测服务及三个防护维度的默认标签集与风险分段。」
- 拦截模式命中响应：使用平台错误码，纳入 F16 网关错误码 i18n 范围（文案清单挂 T17）。

## 8.8.6 状态流转

### 观察模式 / 拦截模式状态转换

- **未启用 → 观察模式**：用户套用推荐配置（或手动启用策略并配置检测维度）后默认进入观察模式。
- **观察模式 → 拦截模式**：用户主动切换，需二次确认。
- **拦截模式 → 观察模式**：用户主动切换回退，可选二次确认。
- **观察模式 / 拦截模式 → 未启用**：关闭护栏策略，需二次确认或关闭所有检测服务开关。

### 观察模式行为

- 命中任何风险（含敏感内容标签）时，仅记录风险结果到调用明细事件，不阻断调用，不改动请求/响应内容。

### 拦截模式行为

- 内容合规、提示词攻击命中高风险 → 阻断调用，返回安全响应；中/低风险 → 仅记录。
- 敏感内容命中已开启标签 → 执行脱敏后继续调用链路。
- AI 安全护栏拦截属于不可重试失败，不触发接入点候选切换。

## 8.8.7 验收标准

### 一、基础行为（AC-8.1 \~ AC-8.3）

- [ ] **AC-8.1** Given 新建一个接入点；When 接入点创建完成；Then 该接入点的 AI 安全护栏状态为「未启用」，调用不经检测服务且无护栏附加延迟。
- [ ] **AC-8.2** Given 接入点 A 启用拦截模式、接入点 B 未启用护栏；When 分别发起调用；Then B 的调用不经过检测服务且无护栏附加延迟，A 的策略变更不影响 B。
- [ ] **AC-8.3** Given 不经接入点的模型直调路径；When 发起模型级别能力授权调用；Then 不触发 AI 安全护栏检测。

### 二、推荐配置与模式（AC-8.4 \~ AC-8.5）

- [ ] **AC-8.4** Given 对接入点套用推荐配置；When 套用完成；Then 策略以观察模式生效，输入/输出检测服务与内容合规、敏感内容、提示词攻击三个防护维度按推荐模板开启。
- [ ] **AC-8.5** Given 接入点策略处于观察模式；When 调用命中任何风险（含敏感内容标签）；Then 调用不被阻断、内容不被改动，调用明细事件记录风险结果。

### 三、拦截模式（AC-8.6 \~ AC-8.8）

- [ ] **AC-8.6** Given 接入点策略处于拦截模式且输入检测服务已开启；When 输入命中提示词攻击高风险（风险分 ≥ 85）；Then 调用被阻断并返回明确安全响应。
- [ ] **AC-8.7** Given 接入点策略处于拦截模式且输出检测服务已开启；When 输出命中内容合规高风险（风险分 ≥ 90）；Then 响应被阻断并返回明确安全响应。
- [ ] **AC-8.8** Given 接入点策略处于拦截模式且敏感内容维度已开启；When 命中已开启敏感内容标签；Then 命中字段被脱敏，调用不中断。

### 四、配置与生效（AC-8.9 \~ AC-8.10）

- [ ] **AC-8.9** Given 关闭某标签或其细分检测范围子项；When 发起调用；Then 对应内容不再命中；全部关闭或未发现异常时检测结果为 `nonLabel`。
- [ ] **AC-8.10** Given 切换标签开关；When 操作完成；Then 在约定生效窗口（2\~5分钟）内变更生效；合规/攻击维度风险分段 v1 只读，编辑操作被拒绝。

### 五、集成与查询（AC-8.11 \~ AC-8.12）

- [ ] **AC-8.11** Given 接入点配置了候选切换且 AI 安全护栏命中拦截；When 护栏拦截发生；Then 不触发接入点候选切换。
- [ ] **AC-8.12** Given 护栏检测结果已写入调用明细事件；When 平台管理员通过 request\_id 查询调用日志；Then 可查看检测方向、防护维度、标签值、风险等级和处置动作。

### 六、检测结果记录（AC-8.13 \~ AC-8.17）

- [ ] **AC-8.13** Given 接入点未启用护栏；When 发起调用；Then 调用明细事件中不产生安全护栏结果字段。
- [ ] **AC-8.14** Given 接入点启用护栏但调用未命中任何标签；When 检测完成；Then 调用明细事件记录 `nonLabel` 为 true，表示本次检测无异常。
- [ ] **AC-8.15** Given 接入点处于观察模式且调用命中风险；When 检测完成；Then 调用明细事件记录检测方向、防护维度、标签值、风险等级，处置动作标记为「记录」。
- [ ] **AC-8.16** Given 接入点处于拦截模式且调用命中高风险被拦截；When 检测完成；Then 调用明细事件记录检测方向、防护维度、标签值、风险等级，处置动作标记为「拦截」。
- [ ] **AC-8.17** Given 接入点处于拦截模式且敏感内容命中已开启标签；When 检测完成；Then 调用明细事件记录检测方向、防护维度、标签值、标签等级，处置动作标记为「脱敏」。

### 七、自托管判官模型（AC-8.18 \~ AC-8.22）

- [ ] **AC-8.18** Given 启用护栏的接入点发起调用；When 调用完成；Then 全部护栏检测无第三方外部服务调用（可断外网验证），判官模型为平台自托管，敏感内容检测全程在网关进程内。
- [ ] **AC-8.19** Given 接入点处于拦截模式且判官模型超时/不可用；When 调用发起；Then 调用 fail-open 放行，调用明细 `guardrail_status=DEGRADED` 且 `degraded=true` 可查。
- [ ] **AC-8.20** Given 启用护栏的接入点发起调用；When 调用完成；Then 判官模型调用不计入租户点数与用量、不出现在租户调用日志，且不递归触发护栏检测。
- [ ] **AC-8.21** Given 接入点处于观察模式；When 发起调用；Then 判官检测异步执行，业务调用关键路径延迟无可感知增加；拦截模式下判官检测同步执行。
- [ ] **AC-8.22** Given 租户查看模型广场或 API Key/接入点可绑定列表；When 浏览列表；Then 判官模型不出现在列表中，租户直调判官 `modelId` 失败。

## 8.8.8 补充约束

- 拦截返回的安全响应错误码纳入 F16 网关错误码 i18n 范围，具体文案清单挂 T17 任务跟踪。
- 安全护栏相关策略与标签配置变更均写审计日志。
- 三个防护维度全部平台自研，不接入第三方检测服务：内容合规/提示词攻击由自托管安全判官模型承担，敏感内容由网关本地规则引擎承担。
- 合规/攻击维度 v1 风险分段只读（三级严重度直接映射：Unsafe=高、Controversial=中），logprob 校准成功后分段编辑作为增强开放。
- 敏感内容 v1 仅包含规则引擎可确定性识别的 7 类结构化信息（个人联系信息、个人身份信息、个人财产信息、个人设备信息、身份鉴别信息、个人地理位置信息、个人基本资料），6 类非结构化信息（个人行为信息、个人关系信息、个人健康生理信息、个人教育工作信息、客户信息、其他个人信息）移出 v1 进后续规划。
- 自由文本个人信息由合规维度「隐私内容」标签承接（语义判定、记录/拦截）。
- 检测服务标签全集以平台标签注册表为准，本节仅固化类别清单、配置模型和代表性标签基线。
- 流式响应截断行为对齐技术约束，本期不做额外定制。
- 图片、文件与多模态检测服务本期不纳入，后续扩展时需重新评估字段与交互方案。
- 安全护栏结果作为调用明细事件的一部分，是用量统计和调用日志查询的事实来源，字段包含检测方向、防护维度、细分检测类别、标签值、风险分（如适用）、风险等级或标签等级、处置动作（记录/脱敏/拦截）。
- 判官模型调用归属运营租户，带 `call_source=GUARDRAIL` 标记，租户侧不可见；token 消耗计平台成本观测口径，不计任何租户限额。
