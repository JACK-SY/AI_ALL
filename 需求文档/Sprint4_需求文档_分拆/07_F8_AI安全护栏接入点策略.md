# F8. AI 安全护栏接入点策略

> 状态：本期。2026-07-15 产品确认：三个防护维度全部平台自研，不接入第三方检测服务——内容合规/提示词攻击由自托管安全判官模型承担，敏感内容由网关本地规则引擎承担；标签体系沿用为平台自有体系。

## 8.8.1 需求说明

### 功能目标

AI 安全护栏作为公共接入点治理能力提供，对经过公共接入点的文本模型调用执行输入/输出安全检测，支持观察与拦截两种运行模式，降低违规内容、敏感信息泄露与提示词攻击风险。

### 触发条件

调用目标为公共接入点 `ep-*` 时，网关解析到公共接入点后加载其护栏策略；业务接入点与不经接入点的模型直调路径本期不经过 AI 安全护栏。

### 前置依赖

- 公共接入点已创建（策略随公共接入点生命周期创建，默认未启用）；
- 自托管安全判官模型（内容合规/提示词攻击维度）已就绪，网关本地规则引擎（敏感内容维度）已实现；
- 网关已集成护栏策略加载与检测调用逻辑（对齐技术边界说明调用链路第 5 步「AI 安全护栏策略通过」）。

### 涉及角色

- 平台管理员：维护公共接入点策略；
- 租户管理员：本期无安全护栏配置权限，业务接入点不提供安全护栏能力。

### 映射产品目标

v1 不默认启用，避免租户无感知被拦截；平台不自动套用任何策略，只提供推荐配置（D-29），暂不支持自定义配置。满足模型调用安全合规要求，为公共接入点提供可观测的安全防护层。

## 8.8.2 产品规则

### 绑定关系

- 一个公共接入点绑定一条接入点策略；不同公共接入点的策略互不影响，业务接入点不绑定安全护栏策略。
- 推荐配置由平台统一维护，公共接入点仅支持套用推荐配置，本期不支持自定义检测维度、标签、阈值、风险等级映射或处置规则。
- 策略随公共接入点生命周期：随公共接入点创建（默认未启用）、随公共接入点删除清理；公共接入点关闭期间策略配置保留。
- 发布公共接入点时可在发布入口勾选「套用 AI 安全护栏推荐配置」，默认不勾选，勾选后以观察模式生效。
- 护栏配置入口保持在接入点配置内，本期不提供独立一级菜单（对齐非目标）。

### 调用链路结合

（对齐技术边界说明调用链路第 5 步「AI 安全护栏策略通过」）

- 调用目标为公共接入点 `ep-*` 时，网关加载该公共接入点的策略；业务接入点与模型直调跳过护栏。
- 输入链路为：原始输入先由本地敏感内容规则执行脱敏，脱敏后的完整内容发送给业务模型，脱敏后内容在超过 32K 时截断至 32K 发送给判官模型，再执行内容合规与提示词攻击检测。
- 输出链路仅由判官模型执行内容合规与提示词攻击检测，输出侧不执行敏感内容脱敏。
- 拦截模式输入命中判官高风险时直接返回安全响应，不消耗上游模型调用；输出命中高风险时在返回调用方前拦截。
- 未启用护栏（或未绑定策略）的接入点跳过检测，不产生护栏附加延迟。
- 不经接入点的模型直调（模型级能力授权路径）本期不经过 AI 安全护栏。

### 状态外显

- 公共接入点详情展示护栏状态：未启用 / 观察模式 / 拦截模式（对齐 8.2.5 公共接入点详情）。

### 策略与运行规则

- 新建接入点默认不启用 AI 安全护栏；未启用护栏的接入点调用跳过检测，无护栏附加延迟。
- 页面引导平台管理员套用推荐配置；推荐配置固定开启输入侧敏感内容脱敏，以及判官模型的输入/输出内容合规与提示词攻击检测。
- 首次套用推荐配置后，默认进入观察模式；用户可从观察模式切换到拦截模式。
- 观察模式下判官模型只记录内容合规与提示词攻击结果，不阻断模型调用；输入侧敏感内容无论观察或拦截模式均先脱敏，脱敏后内容同时提供给判官模型与业务模型，脱敏行为本期仅记录不阻断。
- 拦截模式下，内容合规或提示词攻击命中 `Unsafe` 高风险时阻断调用并返回安全响应，命中 `Controversial` 中风险时只记录。
- 判官模型对输入和输出分别执行内容合规、提示词攻击检测；敏感内容不由判官模型检测，仅由输入侧本地脱敏规则处理。
- 判官模型单次送检最大上下文为 32K；超过 32K 时仅截断判官模型送检内容，业务模型仍接收完整的脱敏后输入。
- 判官模型需结合对话历史识别多轮渐进诱导；在 32K 上下文范围内按协议解析后的对话内容送检。
- 支持 `v1/chat/completions`、`v1/responses` 等文本模型调用协议，各协议执行一致的输入脱敏、输入/输出判官检测、拦截与日志规则。
- 全部检测项关闭，或检测计算后未发现异常时，检测结果返回 `nonLabel`。
- AI 安全护栏拦截属于不可重试失败，不触发接入点候选切换（对齐 8.2.6）。
- 拦截返回的安全响应使用平台错误码，纳入 F16 网关错误码 i18n 范围（文案清单挂 T17）。
- 每次护栏结果写入公共接入点「拦截日志」Tab；拦截调用在用量统计中记为失败调用，具体检测方向、维度、标签、风险等级、处置动作与失败原因在拦截日志中查看。
- 未命中任何标签时记录 `nonLabel`，表示本次检测无异常。
- 平台管理员可在公共接入点详情的「拦截日志」Tab 通过 `request_id` 查询拦截明细。
- 图片、文件与多模态检测服务（AIGC 输入 / 输出图片安全检测、文件实时检测、图文混审等）不在本期范围。
- 原口径中单列的「敏感信息泄露」不再作为独立检测项：个人 / 企业敏感信息识别由敏感内容维度覆盖，泄露与投毒类攻击由提示词攻击维度的「数据泄露与投毒」类别覆盖。
- 本期不引入：恶意 URL 维度、模型幻觉维度（检测服务侧公测中）、自定义检测 Agent、知名 IP 检测 Agent、自定义词库。

### 检测结果与用量记录规则

- 输入或输出被安全护栏拦截时，用量统计将该次公共接入点调用记为失败调用，并计入失败调用数。
- 公共接入点「拦截日志」Tab 仅展示被拦截调用的明细，包括 `request_id`、调用协议、检测方向、防护维度、标签、风险等级、处置动作、失败原因和时间。
- 观察模式命中但未拦截的调用不进入拦截日志；判官模型超时后业务调用成功时不记为失败，也不记为拦截。
- 输入侧敏感内容脱敏后继续调用，不单独记为失败调用；如后续被判官模型拦截，则按拦截失败记录。

## 8.8.3 边界条件与异常场景

### 输入边界

- 判官模型输入最大上下文为 32K；超过 32K 时按协议解析完整对话后截断至 32K 送检，不拒绝业务请求，不截断业务模型接收的脱敏后输入。
- 截断必须采用确定性规则，并在拦截日志中记录已截断标识，便于问题追溯。

### 并发与容量

- 判官模型需支持公共接入点并发调用；应验证目标并发下的吞吐、排队、延迟和超时比例，判官容量不足触发超时时按 fail-open 处理，不得拖垮业务调用链路。

### 降级容灾

- **判官模型超时或不可用**：本次内容合规与提示词攻击检测自动跳过并 fail-open，继续业务调用；公共接入点策略状态保持不变，不切换为观察模式。
- 输入侧敏感内容脱敏规则不随判官模型超时降级，仍先脱敏并将脱敏后的内容提供给业务模型。
- 判官模型恢复后后续请求自动恢复正常检测，无需人工干预。

### 流式响应截断边界

- 流式响应输出检测命中拦截条件时，已发送至调用方的 SSE 片段不可撤回，截断在下一个检测窗口执行；调用方收到的响应为部分内容 + 流结束标记 + 安全响应尾部提示（对齐 T18）

### 配置生效窗口期

- 配置修改对生产环境生效通常需要 2\~5 分钟；此窗口期内，新旧策略可能共存：
  - 请求 A 使用旧策略（编辑前配置），请求 B 使用新策略（编辑后配置），均属正常行为；
  - 编辑操作需在配置页明示生效窗口并提示谨慎操作。

### 存量兼容

- 存量接入点（Sprint 4 之前创建）策略默认为「未启用」，需手动开启并配置；不存在自动迁移或强制激活。

| 标签值                          | 含义      | 标签描述                                                                  |
| ---------------------------- | ------- | --------------------------------------------------------------------- |
| `Direct Prompt Injection`    | 直接提示词注入 | 在输入中直接插入恶意指令，覆盖模型原有规则，诱导生成特定有害内容；支持对模拟对话注入、对立响应注入、道德绑架注入、语种注入等攻击方式的检测 |
| `Indirect Prompt Injection`  | 间接提示词注入 | 通过第三方数据源（在线网页、外部文档等）间接引入恶意指令；支持的攻击方式检测同上                              |
| `Recursive Prompt Injection` | 递归提示词注入 | 提示词对应的输出内容本身包含注入指令，可连锁攻击其它接受该输出作为输入的 LLM 系统                           |
| `Prompt Leaking`             | 提示词泄漏   | 诱导模型泄漏预设的系统提示词                                                        |

其余 5 类（越狱、算力消耗、恶意操作、身份与会话异常、数据泄露与投毒）标签结构相同，标签清单以平台标签注册表为准。判官模型需结合多轮对话上下文识别渐进式诱导。

### 公共接入点拦截日志字段

公共接入点详情新增「拦截日志」Tab，展示被安全护栏拦截的调用明细；用量统计仅将对应调用计为失败。

| 字段     | 说明                                      | 是否本期新增 |
| ------ | --------------------------------------- | :----: |
| 检测方向   | 输入 / 输出                                 |    是   |
| 防护维度   | 内容合规 / 提示词攻击                       |    是   |
| 细分检测类别 | 如「色情内容检测」「个人身份信息」「提示词注入」等                |    是   |
| 标签值    | 平台标签注册表中的英文标签标识 |    是   |
| 风险等级   | 中 / 高            |    是   |
| 处置动作   | 拦截                              |    是   |
| 调用协议 | `v1/chat/completions` / `v1/responses` 等文本协议 | 是 |
| 截断标识 | 判官输入是否因超过 32K 被截断 | 是 |
| 误报标注 | 平台管理员可对拦截记录标记为误报，标注后记录状态变更为误报 | 是 |

## 8.8.5 页面交互

### 操作入口

- 公共接入点发布入口提供「套用 AI 安全护栏推荐配置」，默认不勾选；公共接入点详情展示护栏状态与「拦截日志」Tab。
- 业务接入点创建、编辑和详情页均不展示安全护栏配置、状态或拦截日志入口。

### 交互流程

- 页面只允许套用推荐配置、切换运行模式或关闭护栏，不提供标签、检测范围、阈值、风险等级映射及处置规则的编辑入口。
- 「拦截日志」Tab 支持按 `request_id`、调用协议、检测方向、防护维度和时间筛选被拦截调用。
- 平台管理员可在「拦截日志」Tab 对单条拦截记录执行「标记为误报」操作，标注后记录显示误报状态，便于区分真实拦截与误拦截。
- 公共接入点护栏仅由平台管理员维护。

### 二次确认

- 从观察模式切换到拦截模式需二次确认，提示文案：「切换为拦截模式后，判官模型命中高风险的调用将被阻断。输入中的敏感内容始终先脱敏。确认切换？」
- 关闭整个护栏需二次确认。

### 提示文案

- 推荐配置套用提示：「推荐配置将以观察模式生效，包含输入侧敏感内容脱敏，以及输入/输出内容合规与提示词攻击检测。」
- 拦截模式命中响应：使用平台错误码，纳入 F16 网关错误码 i18n 范围（文案清单挂 T17）；输入或输出命中高风险被拦截时，返回固定安全响应文案：「很抱歉，我无法提供相关内容」。

## 8.8.6 状态流转

### 观察模式 / 拦截模式状态转换

- **未启用 → 观察模式**：平台管理员套用推荐配置后默认进入观察模式。
- **观察模式 → 拦截模式**：用户主动切换，需二次确认。
- **拦截模式 → 观察模式**：用户主动切换回退，可选二次确认。
- **观察模式 / 拦截模式 → 未启用**：关闭护栏策略，需二次确认。

### 观察模式行为

- 判官模型命中内容合规或提示词攻击风险时仅记录，不阻断调用；输入侧敏感内容仍先脱敏。

### 拦截模式行为

- 内容合规、提示词攻击命中高风险 → 阻断调用并返回安全响应；中风险 → 仅记录。
- 输入侧敏感内容在判官模型和业务模型调用前完成脱敏，输出侧不执行敏感内容脱敏。
- AI 安全护栏拦截属于不可重试失败，不触发接入点候选切换。

## 8.8.7 验收标准

### 一、适用范围与推荐配置（AC-8.1 \~ AC-8.4）

- [ ] **AC-8.1** Given 新建公共接入点；When 发布时未勾选推荐配置；Then 护栏状态为「未启用」，调用不执行护栏且无附加延迟。
- [ ] **AC-8.2** Given 创建或查看业务接入点；When 进入创建、编辑或详情页；Then 不展示安全护栏配置、状态或拦截日志入口，调用不执行护栏。
- [ ] **AC-8.3** Given 不经接入点的模型直调路径；When 发起模型级别能力授权调用；Then 不触发 AI 安全护栏。
- [ ] **AC-8.4** Given 发布公共接入点时套用推荐配置；When 发布完成；Then 护栏以观察模式生效，且页面不存在自定义标签、阈值、风险映射或处置规则入口。

### 二、输入脱敏与判官检测（AC-8.5 \~ AC-8.9）

- [ ] **AC-8.5** Given 公共接入点启用护栏；When 输入包含推荐规则可识别的敏感内容；Then 输入先脱敏，脱敏后内容同时提供给判官模型与业务模型，原始敏感内容不提供给两者。
- [ ] **AC-8.6** Given 公共接入点启用护栏；When 判官模型执行检测；Then 判官模型仅对内容合规、提示词攻击两个维度分别执行输入与输出检测，敏感内容不进入判官检测维度。
- [ ] **AC-8.7** Given 护栏处于观察模式；When 判官模型命中中风险或高风险；Then 调用不被阻断，风险结果被记录，输入侧敏感内容仍按规则脱敏。
- [ ] **AC-8.8** Given 护栏处于拦截模式；When 输入或输出命中判官 `Unsafe` 高风险；Then 调用被拦截并返回安全响应，不触发候选切换，用量统计记为失败调用。
- [ ] **AC-8.9** Given 护栏处于拦截模式；When 输入或输出命中判官 `Controversial` 中风险；Then 调用不被拦截，风险结果被记录。

### 三、日志与统计（AC-8.10 \~ AC-8.12）

- [ ] **AC-8.10** Given 输入或输出被安全护栏拦截；When 查询用量统计；Then 该次调用计为失败并计入失败调用数。
- [ ] **AC-8.11** Given 公共接入点存在被护栏拦截的调用；When 平台管理员进入该接入点的「拦截日志」Tab；Then 可按 `request_id` 查看调用协议、检测方向、防护维度、标签、风险等级、处置动作、失败原因和时间。
- [ ] **AC-8.12** Given 判官模型超时但业务调用成功，或输入仅发生敏感内容脱敏；When 查询用量统计和拦截日志；Then 不因超时或脱敏记为失败调用，且不产生拦截记录。

### 四、上下文与降级（AC-8.13 \~ AC-8.16）

- [ ] **AC-8.13** Given 脱敏后输入不超过 32K；When 送判官模型检测；Then 判官模型接收完整脱敏后输入。
- [ ] **AC-8.14** Given 脱敏后输入超过 32K；When 送判官模型检测；Then 按确定性规则截断至 32K 送检，业务模型仍接收完整脱敏后输入，业务请求不因超长被拒绝。
- [ ] **AC-8.15** Given 拦截模式下判官模型超时或不可用；When 发起调用；Then 本次判官检测自动跳过并 fail-open，公共接入点策略状态不变，业务调用继续执行。
- [ ] **AC-8.16** Given 判官模型超时且输入包含敏感内容；When 发起调用；Then 本地脱敏规则仍生效，业务模型只接收脱敏后内容。

### 五、并发、多轮与协议（AC-8.17 \~ AC-8.20）

- [ ] **AC-8.17** Given 多个公共接入点并发调用判官模型；When 并发达到目标测试负载；Then 记录吞吐、延迟、排队和超时比例，超时请求按 fail-open 处理且业务调用链路保持可用。
- [ ] **AC-8.18** Given 前几轮对话内容正常、后续轮次逐步诱导违规输出；When 通过公共接入点连续调用；Then 判官模型结合 32K 范围内的多轮上下文识别渐进诱导并按运行模式处置。
- [ ] **AC-8.19** Given 分别通过 `v1/chat/completions` 与 `v1/responses` 调用公共接入点；When 发送相同测试内容；Then 两种协议均执行一致的输入脱敏、输入/输出判官检测、拦截及日志规则。
- [ ] **AC-8.20** Given 流式与非流式文本调用；When 输出命中高风险；Then 非流式响应在返回前拦截，流式响应按检测窗口停止后续输出并记录拦截。

### 六、成本与模型隔离（AC-8.21 \~ AC-8.22）

- [ ] **AC-8.21** Given 公共接入点启用护栏；When 判官模型完成检测；Then 判官调用不计租户点数、用量和限额，不出现在租户调用日志，且不递归触发护栏。
- [ ] **AC-8.22** Given 租户查看模型广场或 API Key/接入点可绑定列表；When 浏览或尝试直调；Then 判官模型不出现在列表中，直调判官 `modelId` 失败。

## 8.8.8 补充约束

- 拦截返回的安全响应错误码纳入 F16 网关错误码 i18n 范围，具体文案清单挂 T17 任务跟踪。
- 安全护栏仅适用于公共接入点，本期仅支持推荐配置，不支持自定义配置。
- 内容合规与提示词攻击由自托管判官模型承担，敏感内容由输入侧网关本地规则脱敏，不接入第三方检测服务。
- 判官模型最大上下文 32K，超长截断送检，超时自动跳过并 fail-open。
- 敏感内容 v1 仅包含规则引擎可确定性识别的 7 类结构化信息（个人联系信息、个人身份信息、个人财产信息、个人设备信息、身份鉴别信息、个人地理位置信息、个人基本资料），6 类非结构化信息（个人行为信息、个人关系信息、个人健康生理信息、个人教育工作信息、客户信息、其他个人信息）移出 v1 进后续规划。
- 自由文本个人信息由合规维度「隐私内容」标签承接（语义判定、记录/拦截）。
- 判官标签全集以平台标签注册表为准，本节仅固化类别清单和代表性标签基线。
- 流式响应截断行为对齐技术约束。
- 安全护栏拦截在用量统计中记为失败调用，具体明细在公共接入点「拦截日志」Tab 查询。
- 判官模型调用归属运营租户，带 `call_source=GUARDRAIL` 标记，租户侧不可见；token 消耗计平台成本观测口径，不计任何租户限额。
