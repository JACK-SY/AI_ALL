# 非功能需求与收尾

## 9. 非功能需求

| 类别  | 要求                                                                                      |
| --- | --------------------------------------------------------------------------------------- |
| 可用性 | 调用前置校验基于本地状态或缓存；OM、IAM、供应商健康不在每次调用时远程查询                                                 |
| 一致性 | API Key 状态、权限范围、资源关闭失效绑定和离职清理必须主动同步转发侧并清理 Redis 缓存                                      |
| 可审计 | 租户授权范围（授权 / 撤权 / 封禁 / 解封 / 可见范围模式切换，区分动作类型）、API Key 状态变化、离职清理、转发侧同步均记录审计日志；调用被授权层拒绝不写审计 |
| 可扩展 | 查询索引与内容索引独立滚动和扩展；索引拆分与别名切换不阻塞调用主链路                                                      |
| 可恢复 | 内容写入失败不影响调用主链路并保留写入状态；限额等用量累计按 `request_id` 幂等                                          |
| 可运维 | 查询索引按月、内容索引按周滚动；读别名支持跨索引查询；到期整索引删除，长期留存手动备份                                             |
| 安全  | 不展示供应商密钥、内部转发账号、未启用模型操作入口；API Key 明文仅限创建人取回并记录审计（见 F13）                                 |
| 兼容性 | 保留现有基础调用链路，迁移存量 API Key 时需提供兼容或迁移方案                                                     |

## 10. 总体验收清单

### 10.1 授权范围与接入点

- [ ] 平台可把模型和公共接入点授权给租户（进入租户授权范围）；授权即可见可调用，无租户侧开启环节。
- [ ] 公共资源可见范围模式支持 `ALL` / `WHITELIST` + 黑名单叠加（黑名单优先）；存量资源视同 `ALL` 无黑名单，无迁移、行为不变。
- [ ] `WHITELIST` 资源对未授权租户、任意资源对被封禁租户全链路完全不可见；调用被拒统一返回 `RESOURCE_NOT_AUTHORIZED`，封禁事实仅平台侧可见。
- [ ] 平台撤销授权或封禁后已绑定 API Key 保留并统一展示「平台撤销授权」失效原因，重新授权或解封自动恢复；授权 / 撤权 / 封禁 / 解封 / 模式切换写审计。
- [ ] 撤销或封禁公共接入点授权不联动背后候选模型；撤销或封禁模型授权不影响已授权公共接入点内部路由。
- [ ] 业务接入点只能使用本租户授权范围内的模型。
- [ ] 接入点候选为渠道锁定候选；同名 `modelId` 多供应商场景可通过指定模型接入点精确调用。
- [ ] 存量接入点候选完成渠道回填。

### 10.2 凭证与查询

- [ ] 凭证列表支持按创建人筛选，并支持按状态多选筛选（枚举与 F4 / F7 一致）。
- [ ] CC Switch 一键填入可用；明文取回限创建人并记录审计。
- [ ] 用量统计与明细查询支持多维度筛选（含明细 UA / IP、调用形式）。
- [ ] 调用日志区分流式与非流式，非流式 TTFT 显示为 `—`。
- [ ] 普通用户视角明细请求 / 响应原文超阈值被截断可展开。
- [ ] TOP 5 卡片二级钻展携带当前时间范围至明细页。
- [ ] 网关自身错误码按 `Accept-Language` 返回简中 / 繁中 / 英文文案；供应商上游错误透传不翻译。

### 10.3 调用日志与用量

- [ ] 查询索引与内容索引拆分上线，`content_ref` 可追溯原文。
- [ ] 读别名覆盖存量索引和新月度索引，跨月查询可用。
- [ ] 内容索引按周滚动、30 天到期整索引删除；查询索引保留 12 个月。
- [ ] 不同角色看到正确的用量视角。
- [ ] 用户级月度 Token 限额按平台限额启用租户名单和用户配置生效；启用租户内未单独配额度的用户走平台默认额度（500M / 月，可配）；达到限额后前置拒绝且不触发路由切换。
- [ ] 限额用量只累加 `status=SUCCESS`；`RATE_LIMITED` 和 `FAILED` 不计入。
- [ ] 限额用量有 Redis 快速计数和持久化月度用户 Token 账本，重复消费不会重复累计。

### 10.4 治理

- [ ] 离职人员 API Key 在 IAM 同步后由系统自动识别并删除，无需手动确认。
- [ ] AI 安全护栏默认不启用，套用推荐配置后默认观察模式；护栏结果记录到调用明细事件，可在调用日志查询。
- [ ] AI 安全护栏覆盖输入 / 输出两个检测服务，防护维度（内容合规、敏感内容、提示词攻击）与标签可独立开关。
- [ ] 拦截模式下高风险命中被阻断、敏感内容命中被脱敏；护栏结果记录到调用明细事件。

### 10.5 供应商

- [ ] vertex 类型供应商可纳管并调用 Vertex 上的 Gemini / Claude。

> 商业资格验收本期移出，随 Backlog L-02b 独立交付。
