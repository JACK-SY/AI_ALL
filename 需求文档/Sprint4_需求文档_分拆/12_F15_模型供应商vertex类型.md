# F15. 模型供应商 vertex 类型

## 8.15.1 需求说明

模型供应商新增 Google Vertex AI 类型。Go 转发侧已具备 vertex 渠道适配能力（`relay/channel/vertex/`），本期只做产品侧接入，不改网关 relay。

## 8.15.2 产品规则

- 供应商类型枚举新增 `vertex`；目标模型为 Vertex 上的 Gemini 和 Claude，不为其他 Vertex 模型做额外适配。
- 支持连通性测试，结果透传网关校验结论。
- 供应商包含多个模型，连通性测试针对供应商下的模型单独执行。

## 8.15.3 边界条件与异常场景

- **连通性测试失败场景**：连通性测试失败时，表单展示失败原因（超时 / 认证失败 / 网络不可达等枚举），不阻止用户保存供应商配置。保存后模型连通状态为"未测试"，用户可稍后重新测试。
- **SA JSON 格式校验**：前端对粘贴内容做基础 JSON 格式校验（括号匹配、必填字段 `type`/`project_id`/`private_key`/`client_email` 存在性），格式错误时提示"SA JSON 格式不正确，请检查后重试"。
- **Region 为空**：Region 为必填，为空时表单不可提交。
- **Project ID 为空**：Project ID 为必填，为空时表单不可提交。
- **代理地址不可达**：代理地址可选；填写后前端不做连通性预校验，实际连通性由保存后测试或调用行为暴露。

## 8.15.4 字段定义

### 页面展示字段 — vertex 类型供应商凭证表单

| 字段                   | 说明                                                 | 是否本期新增 |  必填 |
| -------------------- | -------------------------------------------------- | :----: | :-: |
| Service Account JSON | 多行文本粘贴框，保存后不回显                                     |    是   |  是  |
| Region               | Vertex AI 区域下拉选择（如 `us-central1`、`europe-west1` 等） |    是   |  是  |
| Project ID           | Google Cloud 项目标识                                  |    是   |  是  |
| 代理地址                 | 可选，前端透传到渠道配置                                       |    是   |  否  |

### 深链字段（CC Switch 一键填入联动）

当 vertex 凭证用于 CC Switch 时，表单额外传递以下字段：

| 字段        | 说明                             | 是否本期新增 |
| --------- | ------------------------------ | :----: |
| endpoint  | 按 vertex 类型取对应的 Vertex AI 网关地址 |    是   |
| region    | 用户选择的 Region                   |    是   |
| projectId | 用户填写的 Project ID               |    是   |

### API 字段

| 字段            | 说明                        | 是否本期新增 |
| ------------- | ------------------------- | :----: |
| channel\_type | 渠道类型枚举新增 `vertex`         |    是   |
| sa\_json      | Service Account JSON 加密存储 |    是   |
| region        | Vertex AI 区域              |    是   |
| project\_id   | Google Cloud 项目标识         |    是   |
| proxy\_url    | 代理地址，可选                   |    是   |

### 数据库字段

| 字段            | 说明                | 是否本期新增 | 存储    |
| ------------- | ----------------- | :----: | ----- |
| channel\_type | 渠道类型枚举新增 `vertex` |    是   | 渠道配置表 |
| sa\_json      | SA JSON 加密密文      |    是   | 渠道凭证表 |
| region        | Vertex AI 区域      |    是   | 渠道配置表 |
| project\_id   | Google Cloud 项目标识 |    是   | 渠道配置表 |
| proxy\_url    | 代理地址              |    是   | 渠道配置表 |

## 8.15.5 页面交互

### vertex 类型供应商创建/编辑表单

- 供应商类型下拉新增 `Google Vertex AI` 选项；选中后表单切换为 vertex 凭证字段。
- SA JSON：多行粘贴框（textarea），粘贴后前端即时校验 JSON 格式，格式错误时输入框下方展示红色提示。
- Region：下拉选择框，选项从平台配置中读取可用区域列表。
- Project ID：文本输入框。
- 代理地址：文本输入框，带 Placeholder "选填，如需代理出网请填写"。
- 保存时 SA JSON 加密写入后端，编辑态不回显原始内容，输入框显示占位符"已保存，出于安全不回显"。
- 连通性测试按钮：点击后调用网关校验接口，展示测试结果（成功 / 失败原因）。

## 8.15.6 状态流转

### 供应商状态

| 供应商状态 | 触发条件      | 目标状态 |
| ----- | --------- | ---- |
| 已启用   | 新建供应商保存成功 | 已启用  |
| 已启用   | 用户点击"禁用"  | 已禁用  |
| 已禁用   | 用户点击"启用"  | 已启用  |

### 模型连通性状态

| 模型连通性状态 | 触发条件         | 目标状态 | 说明           |
| ------- | ------------ | ---- | ------------ |
| 未测试     | 供应商下新增模型保存成功 | 未测试  | 用户可手动发起连通性测试 |
| 未测试     | 用户点击"测试连通性"  | 测试中  | 调用网关校验接口中    |
| 测试中     | 网关校验成功       | 成功   | 展示连通成功       |
| 测试中     | 网关校验失败       | 失败   | 展示失败原因，需重新测试 |

## 8.15.7 验收标准

### 一、供应商创建与安全（AC-15.1 \~ AC-15.2）

- [ ] **AC-15.1** Given 用户在模型供应商创建页面；When 选择 Google Vertex AI 类型并填写 SA JSON、Region、Project ID 后保存；Then 系统创建 vertex 类型供应商，SA JSON 加密存储不回显。
- [ ] **AC-15.2** Given 用户已保存 vertex 供应商配置；When 重新进入编辑页面；Then SA JSON 字段展示占位符"已保存，出于安全不回显"，无法获取原始内容。

### 二、连通性测试（AC-15.3 \~ AC-15.4）

- [ ] **AC-15.3** Given vertex 供应商下模型配置保存成功；When 用户点击"测试连通性"按钮；Then 系统调用网关校验接口，展示"连通成功"。
- [ ] **AC-15.4** Given vertex 供应商下模型配置中 SA JSON 无效或网络不可达；When 用户点击"测试连通性"按钮；Then 系统展示失败原因（超时/认证失败/网络不可达等），不阻止保存。

### 三、配置与校验（AC-15.5 \~ AC-15.7）

- [ ] **AC-15.5** Given vertex 供应商配置了代理地址；When 调用 Vertex AI 接口；Then 请求经代理出网。
- [ ] **AC-15.6** Given 用户在 SA JSON 粘贴框输入非法 JSON 内容；When 内容变更；Then 输入框下方展示红色提示"SA JSON 格式不正确，请检查后重试"。
- [ ] **AC-15.7** Given vertex 供应商已创建；When 用户在该供应商下添加模型；Then 可纳管 Vertex 上的 Gemini 和 Claude 模型。

## 8.15.8 补充约束

- SA JSON 加密存储采用平台统一加密机制，与其他凭证字段一致。
- 连通状态仅在用户手动测试时更新。
- Region 列表从平台配置读取，后续新增区域时前端无需发版。
- 代理地址为前端透传字段，平台不对代理地址做额外校验或转换。
