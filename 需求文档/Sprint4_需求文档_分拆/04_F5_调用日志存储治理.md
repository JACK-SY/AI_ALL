# F5. 调用日志存储治理（ES 索引拆分）

## 8.5.1 需求说明

- **功能目标**：将调用日志的查询字段与内容正文分索引存储，按周期滚动、整索引过期，实现存储治理止血。
- **触发条件**：每次模型调用产生调用日志时触发写入；索引生命周期管理由定时任务自动执行。
- **前置依赖**：Elasticsearch 集群可用；Java 后端索引名配置项（`ModelOcElasticsearchProperties`）可修改；写别名和读别名已创建。
- **涉及角色**：系统（自动写入与过期清理）、平台管理员（手动备份长期内容）。

## 8.5.2 产品规则

- PR-5.1：查询索引按月分割：`kmo_call_log_YYYYMM`，只保存监控 / 检索字段和 `content_ref`，不保存请求 / 响应原文；保留 12 个月。
- PR-5.2：内容索引按周分割：`kmo_call_log_content_YYYYwWW`，保存请求 / 响应原文；默认保留 30 天，到期整索引删除；长期留存由平台管理员按需手动备份。
- PR-5.3：`content_ref` 显式保存内容索引名 + 文档 ID，不靠规则从 `request_id` 推算索引位置；后续调整分割粒度或切换对象存储不影响历史引用。
- PR-5.4：读别名（如 `kmo_call_log_search`）覆盖全部查询索引；存量 `kmo_call_log` 索引不做 reindex，直接作为成员挂入读别名，数据过保留期后整索引删除。
- PR-5.5：写别名（如 `kmo_call_log_write`）按月滚动指向当月查询索引。
- PR-5.6：查询侧（Java 后端）索引名为配置项（`ModelOcElasticsearchProperties`），切换读别名只需修改配置。
- PR-5.7：查询索引保留 `client_ip`、`user_agent` 字段，支撑 F14 的 UA / IP 筛选。
- PR-5.8：内容写入失败不影响调用主链路，调用日志记录调用内容写入状态。

## 8.5.3 边界条件与异常场景

- **别名切换窗口期一致性**：写别名从上月索引切换到新月索引的短暂窗口内，需保证跨月时刻（如每月 1 日 00:00）写入的调用日志不丢失；切换操作应在低峰期执行并在秒级完成，切换期间并发写入的请求通过双重写入或缓冲队列保障一致性。
- **写入路由一致性**：内容索引文档写入后，`content_ref` 中记录的索引名和文档 ID 必须与实际写入位置一一对应；若内容写入失败并重试到下周索引，需更新 `content_ref` 指向重试后实际存储的索引。
- **存量索引兼容**：存量 `kmo_call_log` 索引中的原文字段在读别名查询时不做字段过滤，仍可查询原文；新月度索引不包含原文字段，前端需根据 `content_ref` 是否存在判断是否提供原文查看入口。
- **内容索引过期删除**：整索引删除操作需确认该索引已不在任何活跃写入流程中，避免删除正在写入的索引。

## 8.5.4 字段定义

### 查询索引字段表（`kmo_call_log_YYYYMM`）

| 字段                                                     | 类型      | 说明                                | 是否本期新增 |
| ------------------------------------------------------ | ------- | --------------------------------- | ------ |
| `request_id`                                           | String  | 请求唯一标识                            | 否      |
| `tenant_id` / `user_id` / `api_key_id`                 | String  | 租户 / 用户 / API Key                 | 否      |
| `endpoint_id`                                          | String  | 接入点                               | 否      |
| `modelId` / `providerId`                               | String  | 命中模型 / 供应商                        | 否      |
| `status`                                               | String  | 调用状态                              | 否      |
| `error_code`                                           | String  | 错误码                               | 否      |
| `tokens_total` / `tokens_prompt` / `tokens_completion` | Long    | Token 用量                          | 否      |
| `tokens_cache_read` / `tokens_cache_write`             | Long    | 缓存读写 Token 用量                     | 否      |
| `points`                                               | Decimal | 点数                                | 否      |
| `latency` / `ttft`                                     | Long    | 耗时 / TTFT（ms）                     | 否      |
| `client_ip`                                            | String  | 客户端 IP                            | 否      |
| `user_agent`                                           | String  | 用户代理                              | 否      |
| `om_metering_fields`                                   | Object  | OM 计量字段                           | 否      |
| `safety_guard_result`                                  | Object  | 安全护栏结果                            | 否      |
| `content_ref`                                          | String  | 内容索引名 + 文档 ID                     | 是      |
| `content_write_status`                                 | Enum    | 内容写入状态：SUCCESS / FAILED / SKIPPED | 是      |

### 内容索引字段表（`kmo_call_log_content_YYYYwWW`）

| 字段              | 类型       | 说明       | 是否本期新增 |
| --------------- | -------- | -------- | ------ |
| `request_id`    | String   | 请求唯一标识   | 否      |
| `request_body`  | Text     | 请求原文     | 否      |
| `response_body` | Text     | 响应原文     | 否      |
| `content_size`  | Long     | 内容大小（字节） | 否      |
| `write_time`    | DateTime | 写入时间     | 否      |

## 8.5.5 页面交互

- **操作入口**：调用日志列表页 → 点击某条日志查看详情；详情页"查看原文"按钮。
- **交互流程**：
  1. 调用日志列表页展示查询索引中的字段，不展示请求 / 响应原文。
  2. 点击某条日志进入详情页，若 `content_ref` 存在且 `content_write_status=SUCCESS`，展示"查看原文"按钮；点击后通过 `content_ref` 从内容索引加载原文。
  3. 若 `content_ref` 不存在（存量数据）或 `content_write_status=FAILED`，不展示"查看原文"按钮，展示对应状态提示。
  4. 若内容索引已过期删除，点击"查看原文"后展示"该调用原文已超过保留期限，不可追溯"。
- **二次确认**：无。
- **提示 / 错误文案**：
  - 原文不可用（索引过期）："该调用原文已超过保留期限，不可追溯"
  - 原文加载中："正在加载调用原文..."

## 8.5.6 状态流转

本功能无生命周期状态。索引的创建 / 滚动 / 过期属于生命周期管理策略，由系统定时任务驱动，不涉及业务实体状态流转。

## 8.5.7 验收标准

### 一、索引拆分与读写（AC-5.1 \~ AC-5.2）

- [ ] **AC-5.1** Given 新调用写入；When 查询查询索引；Then 不含请求/响应原文，原文可经 `content_ref` 在内容索引中取到。
- [ ] **AC-5.2** Given 查询侧使用读别名；When 执行跨月时间范围查询；Then 同时命中存量 `kmo_call_log` 数据和新月度索引数据。

### 二、索引滚动与生命周期（AC-5.3 \~ AC-5.5）

- [ ] **AC-5.3** Given 内容索引按周滚动；When 超过 30 天；Then 内容索引可整索引删除；删除后对应调用记录仍可查询，只是原文不可追溯。
- [ ] **AC-5.4** Given 查询索引按月滚动；When 超过 12 个月；Then 查询索引可整索引删除。
- [ ] **AC-5.5** Given 存量 400GB `kmo_call_log` 索引；When 数据过保留期；Then 可一次性删除。

### 三、异常处理与一致性（AC-5.6 \~ AC-5.8）

- [ ] **AC-5.6** Given 内容写入失败；When 调用正常完成；Then 调用不受影响，日志可查见 `content_write_status=FAILED`。
- [ ] **AC-5.7** Given 写别名切换窗口期；When 跨月时刻并发写入；Then 调用日志不丢失且路由到正确索引。
- [ ] **AC-5.8** Given `content_ref` 已记录；When 内容写入重试到不同索引；Then `content_ref` 更新为实际存储索引。

## 8.5.8 补充约束

- 索引滚动与过期策略配置项需支持动态调整，无需重启服务。
- 长期备份流程需提供导出接口，支持按时间范围和租户维度筛选导出。
- 索引拆分上线前需完成灰度验证：先在只读模式运行一段时间，确认查询结果与原索引一致后再切换写入。
