# F2. 接入点渠道锁定与路由意图

## 8.2.1 需求说明

- **功能目标**：将接入点候选从”按模型名选择”升级为”渠道锁定候选”，候选在配置时即绑定具体渠道（模型供应商）+ 模型，`ep-*` 解析后直达该渠道，解决同一 `modelId` 来自多个供应商时无法精确路由的问题。
- **触发条件**：平台管理员或租户管理员创建/编辑接入点时选择路由意图并绑定渠道锁定候选；存量接入点候选需要回填渠道归属。
- **前置依赖**：
  - 纳管模型已注册且至少有一条可用渠道记录；
  - 转发侧已支持渠道锁定解析能力（`endpoint_model_mapping` 新增渠道字段）。
- **涉及角色**：平台管理员（维护公共接入点）、租户管理员/业务负责人（维护业务接入点）。
- **映射产品目标**：消除同名模型多供应商场景下的调用歧义，提升接入点路由的确定性与可控性。

接入点是业务侧稳定调用入口。本期区分公共接入点和业务接入点两类归属类型。公共接入点由运营租户维护，租户侧只读；业务接入点由创建租户维护。

路由意图本期仅提供指定模型（P0，新建接入点默认意图）。

## 8.2.2 产品规则

- 接入点区分两类归属类型：
  - **公共接入点**：由平台管理员维护，面向平台统一提供给多个租户使用的调用入口；租户只能使用和查看摘要，不能编辑路由。
  - **业务接入点**：由租户管理员/业务负责人维护，面向单租户业务系统或场景入口；归属当前租户，只能选择本租户授权范围内的模型。
- 创建或编辑接入点时，必须选择一个路由意图。
- 路由意图类型：
  - **指定模型**（P0默认）：只绑定一条渠道锁定候选（`providerId + modelId`），不做切换或分流。
- 所有候选均为渠道锁定候选，绑定键为 `providerId + modelId`，调用解析后直接使用该渠道，不再按模型名在全部渠道中重新选择。
- `endpoint_model_mapping` 需要新增渠道字段（`providerId`）。
- 候选是否属于同一模型能力由平台管理员人工确认，系统不只靠 `modelId` 字符串自动判断。
- 指定模型意图下，接入点只有一条候选，调用直达该渠道；候选不可用时调用失败，不切换到其他渠道或同名模型。
- 实际命中模型（`providerId + modelId`）和路由意图必须写入调用日志。
- 公共接入点默认模型摘要规则：指定模型意图取唯一候选；其他候选不展示，实际命中模型以调用日志为准。
- 业务接入点已绑定的候选模型被租户管理员关闭或模型不可用后，该候选进入不可用或失效候选状态；系统不自动替换成其他模型。
- 租户管理员需要手动编辑业务接入点路由候选来恢复可调用目标。

## 8.2.3 边界条件与异常场景

- **候选渠道歧义回填场景**：现有接入点候选映射没有渠道归属需要回填；`modelId` 当前只有一个可用渠道时自动回填；存在多渠道歧义的候选输出清单，由平台管理员逐条确认后回填。[待确认:T-01] 歧义候选清单的输出格式与确认界面交互方式。
- **缓存失效窗口**：渠道绑定信息缓存与数据库不一致的窗口期内，调用可能解析到已失效渠道；需确认缓存刷新机制与最大延迟容忍。[待确认:T-03] 缓存刷新延迟上限与降级策略。
- **并发竞态**：平台管理员正在编辑接入点候选的同时有并发调用进入，需保证路由解析使用一致的候选快照；编辑保存应采用乐观锁或版本号机制防止覆盖。[待确认:T-04] 编辑并发控制策略。
- **存量兼容**：未完成渠道回填的存量接入点在本期上线后调用行为需保持兼容；回填完成前按原模型名路由逻辑继续运行，直到管理员确认回填或系统标记过期。[待确认:T-05] 存量接入点回填过渡期截止策略。

## 8.2.4 字段定义

### 接入点候选渠道字段表（endpoint_model_mapping）

| 字段名 | 类型 | 说明 | 是否本期新增 |
|--------|------|------|-------------|
| id | String | 候选映射记录主键 | 否 |
| endpointId | String | 接入点 ID | 否 |
| modelId | String | 模型 ID（如 glm-5.1） | 否 |
| providerId | String | 模型供应商 ID | **是** |
| providerName | String | 模型供应商名称 | **是** |
| isActive | Boolean | 候选是否可用 | 否 |
| sortOrder | Integer | 候选排序序号 | 否 |
| createdAt | DateTime | 创建时间 | 否 |
| updatedAt | DateTime | 最后更新时间 | 否 |

### API Key 候选选择器列结构表

| 列名 | 对应字段 | 说明 | 是否本期新增 |
|------|----------|------|-------------|
| 模型 code | modelId | 模型 ID 展示 | 否（保留原有列） |
| 供应商名称 | providerName | 候选背后供应商名称 | **是**（替换原”模型名称”列） |
| 模型规格 | modelSpec | 模型规格摘要 | 否 |
| 勾选框 | providerId + modelId | 勾选精确到 providerId + modelId | **是**（勾选粒度变更） |

### 数据库/索引字段表

| 字段名 | 所属表/索引 | 类型 | 说明 | 是否本期新增 |
|--------|-----------|------|------|-------------|
| providerId | endpoint_model_mapping | String | 渠道锁定：模型供应商 ID | **是** |
| providerId | endpoint_model_mapping 复合索引 | — | (endpointId, providerId, modelId) 复合唯一索引 | **是** |
| routingIntent | endpoint | Enum | 路由意图：designated | **是** |

## 8.2.5 页面交互

### 操作入口

- 平台管理员：接入点管理 → 创建/编辑公共接入点 → 选择路由意图 → 绑定渠道锁定候选
- 租户管理员：接入点管理 → 创建/编辑业务接入点 → 选择路由意图 → 绑定渠道锁定候选（限本租户授权范围）

### API Key 编辑器候选选择器（优化项）

API Key 编辑弹窗的”模型直连（高级）”标签下的候选选择器需要显式展示供应商信息，避免同名 `modelId` 无法区分：

- 列结构调整：模型 code / 供应商名称 / 模型规格 —— 当前实现的”模型名称”列与”模型 code”实为同一 `modelId`（截断展示），冗余；本期将该列替换为”供应商名称”。
- 供应商名称取候选背后的 `providerName`；同一 `modelId` 来自不同供应商时，逐行独立展示，勾选精确到 `providerId + modelId`。
- 后台保存和校验以 `providerId + modelId` 为准，与 CONTEXT.md「API Key 模型绑定」一致。
- 搜索框可按模型 code 和供应商名称匹配。

### 租户侧接入点列表与操作

- 租户侧接入点管理使用一个列表展示公共接入点和业务接入点，通过类型字段和筛选区分。
- 公共接入点在租户侧只读，业务接入点由本租户维护。
- 公共接入点详情展示调用方式、状态、AI 安全护栏状态、调用日志入口、路由意图和当前默认模型摘要。
- 公共接入点当前默认模型摘要展示 `modelId`、`providerName` 和路由意图。
- 业务接入点路由候选选择器按模型展示，候选项只显示模型 ID 和供应商来源。
- 业务接入点路由候选选择器不展示供应商账号、服务地址、成本、内部转发配置、可用状态或能力摘要。
- 公共接入点详情租户不可见：完整候选列表、候选数量、供应商来源明细/供应商账号/服务地址/成本/内部转发配置。

### 二次确认规则

- 删除接入点需二次确认。
- 解绑候选（指定模型意图下唯一的候选）需二次确认并提示”候选解绑后该接入点调用将失败”。[待确认:T-06]

### 提示/错误文案

- 候选不可用时，接入点状态标记为不可用，详情页提示”当前候选不可用，调用将失败”。
- 存量候选存在渠道歧义时，提示”以下模型的渠道归属需要确认”并列出歧义清单。

## 8.2.6 状态流转

本功能无生命周期状态。接入点本身的启用/禁用状态和候选可用性属于接入点管理功能范畴，渠道锁定与路由意图功能范围内不定义独立的状态机。

## 8.2.7 验收标准

### 一、接入点基础操作（AC-2.1 ~ AC-2.3）

- [ ] **AC-2.1** Given 平台管理员登录管理后台；When 创建或编辑公共接入点并保存；Then 可成功创建或更新公共接入点。
- [ ] **AC-2.2** Given 租户管理员登录租户侧；When 创建或编辑本租户业务接入点并保存；Then 可成功创建或更新业务接入点。
- [ ] **AC-2.3** Given 租户管理员查看公共接入点；When 尝试编辑路由和候选模型配置；Then 操作被拒绝，界面不可编辑或按钮不可点击。

### 二、渠道锁定与路由意图（AC-2.4 ~ AC-2.8）

- [ ] **AC-2.4** Given 两个供应商均提供同名模型 `glm-5.1`；When 分别创建指定模型意图接入点并绑定不同供应商的渠道锁定候选；Then 调用 `model=ep-xxx` 时稳定命中对应供应商的渠道。
- [ ] **AC-2.5** Given 接入点路由意图为指定模型；When 该唯一候选不可用时；Then 调用失败，不切换到其他渠道的同名模型。
- [ ] **AC-2.6** Given 创建新接入点；When 进入路由意图选择步骤；Then 默认路由意图为指定模型。
- [ ] **AC-2.7** Given API Key 编辑器"模型直连（高级）"候选选择器；When 打开选择器；Then 原"模型名称"列已替换为"供应商名称"列，同名 `modelId` 多供应商逐行独立展示，勾选粒度精确到 `providerId + modelId`。
- [ ] **AC-2.8** Given 存量接入点候选存在渠道归属缺失；When 执行渠道回填；Then 单渠道候选自动回填完成；多渠道歧义候选输出确认清单，管理员逐条确认后完成回填。

### 三、租户侧权限与展示（AC-2.9 ~ AC-2.11）

- [ ] **AC-2.9** Given 租户管理员创建或编辑业务接入点；When 选择候选模型；Then 只能选择本租户授权范围内的模型。
- [ ] **AC-2.10** Given 业务接入点路由候选选择器；When 查看候选项；Then 只展示模型 ID 和供应商来源，不展示可用状态和能力摘要。
- [ ] **AC-2.11** Given 租户侧查看公共接入点详情；When 页面加载完成；Then 展示路由意图、默认模型摘要和调用契约信息，不展示完整候选列表、候选数量和供应商配置。

## 8.2.8 补充约束

- **存量候选回填歧义确认规则**：多渠道歧义候选必须由平台管理员逐条确认后才能完成回填；系统不自动选择渠道归属，确认过程需留痕（确认人、确认时间、所选渠道），以保证追溯性。
- **候选不可用不自动切换**：已散落在产品规则中（指定模型意图候选不可用不切换），本约束不再重复。
