# F13. CC Switch 一键填入

## 8.13.1 需求说明

大模型接入凭证支持一键填入 cc-switch 本地应用：平台把 API Key、网关地址和模型选择拼装为 `ccswitch://` 深链，浏览器拉起 cc-switch 完成 provider 导入。深链格式：`ccswitch://v1/import?resource=provider&app=<claude|codex|gemini>&name=<名称>&endpoint=<网关地址>&apiKey=<明文密钥>`。

## 8.13.2 产品规则

- 入口：**API Key 管理列表的操作列行操作按钮**；仅创建人可见可用。
- 对话框字段：应用（Claude / Codex / Gemini）、名称、主模型（必填）及应用对应的模型映射（如 Claude 的 Haiku / Sonnet / Opus 模型）。
- 模型下拉选项取该 Key 权限范围内的可调用目标，含 `ep-*` 接入点；指定模型接入点可直接作为 Claude Code 等工具的模型使用。
- endpoint 取平台网关对外地址，按所选应用给出对应协议入口。

## 8.13.3 边界条件与异常场景

- **cc-switch 未安装场景**：浏览器尝试打开 `ccswitch://` 深链后无法拉起本地应用，系统不额外拦截，由浏览器展示默认的"无法打开此链接"提示。对话框底部增加静态提示文案"请确保已安装 cc-switch 客户端"。
- **深链参数过长**：名称超长导致完整深链超过浏览器 URL 长度限制（约 2048 字符）时，系统在对话框生成深链失败并提示"参数过长，请缩短名称后重试"。
- **API Key 已禁用 / 已删除**：对已禁用或已删除的 API Key，列表行操作中不展示"一键填入"按钮。
- **模型映射为空**：用户未选择任何模型映射时，深链中不包含模型映射参数；用户可在 cc-switch 中手动配置。
- **浏览器安全策略拦截**：部分浏览器可能拦截非 HTTPS 深链跳转，系统无法绕过此限制，对话框提示"如无法跳转，请检查浏览器设置或手动复制配置"。

## 8.13.4 字段定义

### 页面展示字段 — 一键填入对话框

| 字段 | 说明 | 是否本期新增 |
|------|------|:----------:|
| 应用 | 下拉选择：Claude / Codex / Gemini | 是 |
| 名称 | Provider 显示名称，文本输入，默认取 API Key 名称 | 是 |
| 主模型 | 下拉选择，必填，取该 Key 权限范围内可调用模型 | 是 |
| 模型映射 | 根据所选应用展示对应映射字段（如 Claude: Haiku/Sonnet/Opus） | 是 |
| 网关地址 | 只读展示，按所选应用从平台配置取 | 是 |
| 提示文案 | "请确保已安装 cc-switch 客户端" 静态提示 | 是 |

### 页面展示字段 — 操作入口

| 入口位置 | 说明 | 是否本期新增 |
|----------|------|:----------:|
| 凭证列表行操作 | 操作列下拉菜单中"填入 cc-switch"选项 | 是 |

### 深链字段

| 字段 | 说明 | 是否本期新增 |
|------|------|:----------:|
| resource | 固定值 `provider` | 是 |
| app | 枚举：`claude` / `codex` / `gemini` | 是 |
| name | 用户输入的 Provider 名称 | 是 |
| endpoint | 平台网关对外地址 | 是 |
| apiKey | API Key 明文（需事后取回） | 是 |
| model | 主模型标识 | 是 |
| modelMapping | 应用对应的模型映射键值对，URL 编码 | 是 |

### API 字段

| 字段 | 说明 | 是否本期新增 |
|------|------|:----------:|
| api_key_plaintext | API Key 明文取回接口返回 | 是（D-46 决策） |

### 数据库字段

本功能不新增数据库字段。明文取回能力依赖现有加密存储机制，审计日志走通用审计表。

## 8.13.5 页面交互

### 凭证列表行操作

- 每行凭证的操作列（行操作按钮 / 下拉菜单）中增加"填入 cc-switch"选项。
- 仅当当前用户为该 API Key 创建人时展示此选项；非创建人不可见。
- 已禁用 / 已删除状态的 API Key 不展示此选项。
- 点击后打开一键填入对话框，行为同上。

### 一键填入对话框交互

- 应用切换后，模型映射字段动态更新为对应应用的映射模型列表。
- 主模型下拉选项来自该 Key 权限范围内可调用的模型 + `ep-*` 接入点。
- 网关地址只读，按应用自动匹配平台配置的对应协议入口。
- 底部静态提示："请确保已安装 cc-switch 客户端"。
- "生成并跳转"按钮点击后，系统取回 API Key 明文，拼装完整深链并跳转；若取回失败则提示"密钥取回失败，请稍后重试"。

### 手动复制兜底

- 对话框同时提供"复制配置"按钮，将关键参数（endpoint、apiKey、模型等）以结构化文本形式复制到剪贴板，供用户手动粘贴。

## 8.13.6 状态流转

本功能不涉及独立状态流转。操作入口的可见性受 API Key 生命周期状态约束：

| API Key 状态 | 列表行操作入口 |
|-------------|:----------:|
| 生效中 | 可见（仅创建人） |
| 已禁用 | 不可见 |

## 8.13.7 验收标准

### 一、入口与权限（AC-13.1 ~ AC-13.2）

- [ ] **AC-13.1** Given 创建人在凭证列表查看自己创建的启用状态 Key；When 点击该行操作列的"填入 cc-switch"选项；Then 对话框打开，预填名称和默认应用，生成深链可拉起 cc-switch 并成功导入。
- [ ] **AC-13.2** Given 非创建人（含租户管理员）查看凭证列表；When 浏览任一非自己创建的 Key 行操作；Then 操作列中不展示"填入 cc-switch"选项。

### 二、操作与权限（AC-13.3 ~ AC-13.5）

- [ ] **AC-13.3** Given 用户通过一键填入功能取回 API Key 明文；When 系统执行明文取回；Then 产生审计日志，记录操作人、时间和凭证标识。
- [ ] **AC-13.4** Given 用户在一键填入对话框中展开主模型下拉；When 查看可选模型列表；Then 列表包含该 Key 可调用的模型和 `ep-*` 接入点。
- [ ] **AC-13.5** Given 创建人查看已禁用或已删除状态的 API Key；When 查看该行操作列；Then 不展示"填入 cc-switch"选项。

### 三、异常与提示（AC-13.6 ~ AC-13.7）

- [ ] **AC-13.6** Given 用户点击"生成并跳转"后 cc-switch 未安装；When 浏览器无法拉起深链；Then 用户看到浏览器默认提示，对话框已有"请确保已安装 cc-switch 客户端"静态文案。
- [ ] **AC-13.7** Given 用户输入的名称过长导致深链超过 2048 字符；When 点击"生成并跳转"；Then 系统提示"参数过长，请缩短名称后重试"。

## 8.13.8 补充约束

- **安全口径调整**：本能力要求事后取回 API Key 明文，安全口径由"仅创建时展示一次"调整为"API Key 明文可取回"：明文仅限创建人本人取回，其他成员（含租户管理员）不可取回；每次取回记录审计日志（操作人、时间、凭证）。此调整归入本节补充约束，不再单独列节。
- 深链中的 `apiKey` 参数为明文传输，安全性由 HTTPS 保障；`ccswitch://` 协议注册由 cc-switch 客户端安装时完成，平台不控制协议注册行为。
- 网关地址从平台系统配置中读取，不由用户手动输入；不同应用可能对应不同协议入口（如 Claude 使用 OpenAI 兼容协议、Gemini 使用原生 Vertex 协议）。
- 一键填入功能本期仅支持桌面端浏览器环境；移动端浏览器对深链协议的支持程度不一，本期不做适配。
- 模型映射参数在深链中以 JSON 结构体 URL 编码传递；cc-switch 侧解析规则由 cc-switch 版本定义，平台仅按约定格式生成。
