# F4. 转发侧租户账号与 API Key 同步

> 状态：✅ 已完成（Sprint 4 迭代中途交付）。以下内容作为已实现行为基线保留。

## 8.4.1 需求说明

- **功能目标**：将模型平台租户维度映射到 Go 转发侧租户账号，使 API Key 状态与授权变更能实时同步到转发侧，保证调用链路一致性。
- **触发条件**：租户首次创建大模型接入凭证；API Key 状态或授权范围发生变化；资源关闭失效绑定产生或恢复；离职资源清理。
- **前置依赖**：Go 转发侧登记接口可用；模型平台数据库可写入 `tenant_id → relay_user_id` 映射；Redis 缓存可访问。
- **涉及角色**：平台管理员（登记租户账号）、系统（自动同步）。

## 8.4.2 产品规则

- PR-4.1：租户首次创建大模型接入凭证时，按需创建转发侧租户账号。
- PR-4.2：租户手动登记时需经过接口验证——模型平台向 Go 转发侧调用登记接口注册租户账号，登记接口返回成功方视为有效；接口调用失败时展示失败提示，不允许跳过验证直接创建。
- PR-4.3：模型平台保存 `tenant_id → relay_user_id` 映射。
- PR-4.4：同租户后续 API Key 复用同一个 `relay_user_id`。
- PR-4.5：转发侧租户账号是技术账号，不等同于模型平台真实用户。
- PR-4.6：API Key 创建人、权限、离职状态和审计仍由模型平台维护。
- PR-4.7：API Key 状态或授权变化后，模型平台主动同步 Go 转发侧 token。
- PR-4.8：同步完成或同步失败后，都要更新模型平台侧同步状态。
- PR-4.9：同步后主动清理 Redis 缓存，不等待自然过期。
- PR-4.10：全部资源权限的 API Key 在 Go 侧不得固化模型或接入点白名单，应使用关闭白名单限制或等价动态策略。

### 同步场景表

| 场景 | 同步动作 |
| ---- | ---- |
| 创建 API Key | 创建或复用转发侧租户账号，创建 Go token |
| 禁用 API Key | Go token 置为不可用，清理缓存 |
| 启用 API Key | Go token 置为可用，清理缓存 |
| 删除 API Key | 删除或禁用 Go token，清理缓存 |
| 权限范围变化 | 更新 Go token 有效权限，清理缓存；部分资源权限手动增删模型或接入点属于权限范围变化 |
| 资源关闭失效绑定产生 | 从 Go token 有效权限中移除对应目标，清理缓存 |
| 资源关闭失效绑定恢复 | 同一资源重新授权 / 解封并恢复有效后，将对应目标加入 Go token 有效权限，清理缓存 |
| 离职资源清理 | 同步禁用或删除状态，清理缓存 |

## 8.4.3 边界条件与异常场景

- **登记接口失败**：Go 转发侧登记接口返回非成功或网络超时时，前端展示”转发侧租户账号登记失败，请稍后重试”提示；模型平台不创建 `relay_user_id` 映射记录，不生成 API Key。
- **Redis 缓存清理失败**：同步操作本身成功但 Redis 清理失败时，记录平台侧异常日志，标记该条同步记录为”缓存清理未完成”；后续通过补偿任务或下一次同步触发时重新清理。
- **并发竞态**：同一租户首个 API Key 创建存在并发请求时，只有一个请求成功登记并写入映射，其余请求检测到已有映射后复用 `relay_user_id`；若并发写入映射发生冲突，以登记接口首次返回成功的记录为准。
- **存量兼容**：Sprint 4 之前已存在的 API Key 仍按原逻辑运行；本期新建或变更的 API Key 走新同步流程。

## 8.4.4 字段定义

### 页面展示字段表

| 字段 | 说明 | 是否本期新增 |
| ---- | ---- | ---- |
| 转发侧同步状态 | 展示同步中 / 同步成功 / 同步失败 | 是 |
| 同步失败原因 | 同步失败时展示具体错误信息 | 是 |

### API 字段表

| 字段 | 类型 | 说明 | 是否本期新增 |
| ---- | ---- | ---- | ---- |
| `relay_user_id` | String | 转发侧租户账号 ID | 否 |
| `sync_status` | Enum | 同步状态：SYNCING / SUCCESS / FAILED | 是 |
| `sync_error_message` | String | 同步失败原因 | 是 |
| `last_sync_time` | DateTime | 最近同步时间 | 是 |

### 数据库 / 索引字段表

| 字段 | 表 | 说明 | 是否本期新增 |
| ---- | ---- | ---- | ---- |
| `relay_user_id` | 租户表 | 转发侧租户账号 ID | 否 |
| `sync_status` | API Key 表 | 同步状态 | 是 |
| `sync_error_message` | API Key 表 | 同步失败原因 | 是 |
| `last_sync_time` | API Key 表 | 最近成功同步时间 | 是 |

## 8.4.5 页面交互

- **操作入口**：API Key 详情页 → 同步状态标签；租户首次创建凭证时的创建向导第二步。
- **交互流程**：
  1. 租户首次创建凭证时，系统自动调用登记接口；成功后展示”转发侧租户账号已登记”，继续后续步骤。
  2. API Key 状态或权限变更后，系统自动触发同步；API Key 详情页刷新后展示最新同步状态。
  3. 同步失败时，状态标签显示红色”同步失败”，hover 展示失败原因；管理员可点击”重试同步”按钮手动触发。
- **二次确认**：无（同步为系统自动行为，重试同步无需二次确认）。
- **提示 / 错误文案**：
  - 登记失败：”转发侧租户账号登记失败，请稍后重试”
  - 同步失败：”同步失败：{具体原因}，可点击重试”
  - 同步成功：状态标签显示绿色”已同步”

## 8.4.6 状态流转

API Key 同步状态无独立生命周期，状态值随同步操作实时更新（SYNCING → SUCCESS / FAILED），不适用 Mermaid 状态图。同步状态与 API Key 自身生命周期（启用 / 禁用 / 删除）关联但非驱动关系。

## 8.4.7 验收标准

### 一、租户账号登记（AC-4.1 ~ AC-4.2）

- [ ] **AC-4.1** Given 租户首次创建大模型接入凭证；When 登记接口返回成功；Then 系统创建转发侧租户账号并保存 `relay_user_id` 映射。
- [ ] **AC-4.2** Given 租户已有 `relay_user_id` 映射；When 该租户再次创建 API Key；Then 新 API Key 复用同一 `relay_user_id`。

### 二、同步操作（AC-4.3 ~ AC-4.7）

- [ ] **AC-4.3** Given API Key 已禁用；When 转发侧同步完成；Then 使用该 API Key 的调用立即失败。
- [ ] **AC-4.4** Given API Key 已删除；When 转发侧同步完成；Then Go token 和 Redis 缓存均被清理。
- [ ] **AC-4.5** Given 资源关闭失效绑定产生；When 转发侧同步完成；Then Go token 有效权限被更新，Redis 缓存被清理。
- [ ] **AC-4.6** Given 资源关闭失效绑定恢复；When 转发侧同步完成；Then Go token 有效权限被更新，Redis 缓存被清理。
- [ ] **AC-4.7** Given API Key 持有全部资源权限；When 租户新增启用资源；Then 不需要逐把更新 Go token 即可调用新资源。

### 三、异常处理（AC-4.8 ~ AC-4.10）

- [ ] **AC-4.8** Given 同步操作执行；When Go 转发侧返回失败或超时；Then API Key 详情页展示同步失败状态和失败原因。
- [ ] **AC-4.9** Given 登记接口调用失败；When 租户尝试创建首个接入凭证；Then 前端展示”转发侧租户账号登记失败，请稍后重试”且不创建映射记录。
- [ ] **AC-4.10** Given Redis 缓存清理失败；When 同步操作本身成功；Then 记录异常日志并标记”缓存清理未完成”，后续补偿任务重新清理。

## 8.4.8 补充约束

- 同步操作必须保证幂等性，同一 API Key 的多次同步请求产生一致结果。
- 转发侧登记接口超时阈值建议不超过 10 秒，超时视为失败。
- 并发登记场景下不允许创建重复的 `relay_user_id` 映射记录。
