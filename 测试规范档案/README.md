# 测试规范档案

> 版本：v2.0 | 生效日期：2026-07-17

本目录包含项目级测试规范体系，适用于项目内所有测试活动和自动化流程。

## 目录结构

```
测试规范档案/
├── 总纲与框架/           # 规范体系总览与基础框架
│   └── 测试规范总纲.md
├── 需求管理/             # 需求解析与追溯相关规范
│   ├── 需求解析规则规范.md
│   └── 需求追溯矩阵规范.md
├── 用例设计/             # 测试用例设计方法与质量控制
│   ├── 测试设计方法规范.md
│   ├── 测试关注点与历史问题规范.md
│   └── 优先级与回归分类规范.md
└── 流程控制/             # 测试流程与文档生成规范
    ├── 需求文档生成规范.md
    ├── 交互模式与流程规范.md
    └── 技能已知问题与改进追踪.md
```

## 规范分类说明

| 分类 | 文件 | 用途 |
|------|------|------|
| **总纲与框架** | 测试规范总纲.md | 规范体系总览、术语定义、执行要求 |
| **需求管理** | 需求解析规则规范.md | 需求ID提取、场景流识别、术语消歧 |
| **需求管理** | 需求追溯矩阵规范.md | 双向追溯、覆盖率计算、孤立项检测 |
| **用例设计** | 测试设计方法规范.md | 设计方法定义（EP/BVA/ST/EG）、测试关注点分类 |
| **用例设计** | 测试关注点与历史问题规范.md | 历史问题清单、用例生成后置检查 |
| **用例设计** | 优先级与回归分类规范.md | 优先级推导规则、回归测试分类 |
| **流程控制** | 需求文档生成规范.md | PRD文档格式、验收标准规范 |
| **流程控制** | 交互模式与流程规范.md | 测试用例生成技能的交互流程 |
| **流程控制** | 技能已知问题与改进追踪.md | 技能缺陷记录与改进状态 |

## 规范读取机制

| 任务类型 | 必须读取的规范 | 可选读取的规范 |
|---------|---------------|---------------|
| 需求文档生成 | 测试规范总纲、需求文档生成规范 | 需求解析规则规范、测试关注点与历史问题规范 |
| 需求解读/测试范围识别 | 需求解析规则规范 | 需求追溯矩阵规范 |
| 测试分析/风险评估 | 测试设计方法规范、优先级与回归分类规范、测试关注点与历史问题规范 | 需求解析规则规范 |
| 测试用例设计/生成 | 测试设计方法规范、测试规范总纲、测试关注点与历史问题规范 | 全部规范 |
| 测试用例评审 | 测试规范总纲、需求追溯矩阵规范 | 测试设计方法规范、测试关注点与历史问题规范 |
| 测试用例维护/交付 | 测试规范总纲、需求追溯矩阵规范 | 需求解析规则规范 |

## 执行要求

1. 任务开始前必须读取对应专项规范
2. 输出前必须对照规范自检
3. 规范间冲突时以《测试规范总纲》和专项规范为准
4. 所有规范均为项目级统一标准，适用于所有测试活动和自动化流程
