## 2. 用例生成后置检查清单

测试用例生成完成后，必须对照以下清单逐项检查是否覆盖：

### 2.1 异常场景检查

- [ ] 输入异常：非法输入、格式错误、必填字段、类型错误
- [ ] 流程异常：跳过步骤、中途中断、页面刷新、前进后退、重复提交
- [ ] 环境异常：网络波动、网络中断、低速网络、服务器负载高、数据库连接失败
- [ ] 依赖异常：第三方服务故障、依赖数据缺失、接口超时
- [ ] 页面组件操作异常：防抖处理、通配符转义、并发请求时序、组件状态同步

### 2.2 边界值检查

- [ ] 数据边界：最小值、最大值、临界值、空值
- [ ] 时间边界：活动开始/结束时间、超时未操作、日期切换
- [ ] 业务边界：库存边界、数量限制、金额边界、频次限制

### 2.3 安全性检查

- [ ] 权限控制：角色权限、越权操作、租户隔离、登录状态
- [ ] 数据安全：敏感数据加密、SQL注入、XSS攻击、CSRF攻击
- [ ] 操作安全：二次确认、操作审计、操作限制

### 2.4 业务一致性检查

- [ ] 功能联动：跨页面联动、跨功能联动、状态转换
- [ ] 数据一致性：多端一致性、列表/详情一致、操作后一致性
- [ ] 业务规则合规性：ID唯一性、格式合规、不可重复
- [ ] 业务异常场景：模型下线、授权撤销、资源失效

### 2.5 历史问题检查（强制）

- [ ] **HQ-001**：URL输入处理，无死循环
- [ ] **HQ-002**：搜索通配符转义
- [ ] **HQ-003**：低速网络前端时序控制
- [ ] **HQ-004**：路由/组件复用状态同步
- [ ] **HQ-005**：必填字段校验失败后重新提交
- [ ] **HQ-006**：防抖处理与并发请求时序

## 5. 更新机制

本规范应根据项目进展和新发现的问题定期更新：

1. **新增问题**：每次发现新的高风险问题后，应在第1节「历史问题重点关注清单」中添加记录，分配新编号（HQ-007、HQ-008...）
2. **问题关闭**：当某个历史问题已通过代码修复并验证后，应标记为已解决并记录修复版本
3. **规范迭代**：每两个Sprint后回顾一次，根据实际情况调整检查清单
