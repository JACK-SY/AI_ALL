## 1. 业务域知识

### 1.1 核心实体

#### 业务线

- **企业 AI 模型中心业务线**：鼎捷数智旗下 AI 模型管理与服务平台
- **MOP 业务线**：模型运营平台（Model Operation Platform），接入点所属应用

#### 产品 / 服务

- **模型广场服务**：提供丰富模型资源，支持精调、快捷API 接入、体验
- **模型体验服务**：零代码快速在线调用模型，验证效果
- **接入点服务**：业务侧可见的 "模型" 的抽象，一次创建，业务代码零感不动
- **模型训练服务**：集中管理用户训练调试的模型，支持自定义模型算法、参数、数据集
- **模型微调服务**：集中管理用户微调的模型，支持快速发布、体验和删除
- **模型评测服务**：集中管理模型评测任务与评测标准，支持创建评测任务、配置评测标准
- **算法库服务**：CV、NLP、TSA 等标准化算法包的上架管理
- **数据集管理服务**：集中管理训练集与评测集，支持本地上传或 API 接入
- **数据预处理服务**：集中管理数据链与数据预处理任务，支持基于模板创建数据链
- **数据标注服务**：支持文本、图像、音频、时序多数据类型的标注与结果导出
- **模型库服务**：集中管理第三方上架模型，支持筛选、部署、发布与下线
- **模型供应商管理服务**：管理多供应商接入配置（10家供应商）、多API Key管理、聚合策略
- **Embedding/Rerank服务**：支持/v1/embeddings、/v1/rerank向量能力接口
- **IndepthAI对接服务**：外接/微调模型调用、租户自动绑定
- **模型定价服务**：定价配置、阶梯计费、档位管理
- **租户用量监控服务**：菜单权限、数据统计、排行榜
- **用量统计服务**：模型调用用量与消费统计分析
- **API Key 管理服务**：模型服务访问凭证管理

#### 业务功能模块

- **大模型 LLM 模块**：大语言模型管理与应用
- **小模型 ML 模块**：传统机器学习模型管理与应用
- **智能体（Agent）模块**：支持智能体场景的模型能力
- **工具调用模块**：支持工具调用场景的模型能力
- **深度思考模块**：支持深度思考推理的模型能力

#### 业务流程节点

> 说明：端到端测试参考依据

- **模型上架流程**：第三方模型接入 → 模型部署 → 模型发布 → 模型广场展示
- **接入点创建流程**：接入点配置 → 模型关联 → 业务集成
- **模型训练流程**：数据准备 → 算法选择 → 参数配置 → 训练执行 → 模型上架
- **模型微调流程**：基座模型选择 → 训练数据准备 → 微调配置 → 微调执行 → 模型发布
- **模型评测流程**：评测集准备 → 评测标准配置 → 评测任务创建 → 评测执行 → 结果分析
- **数据处理流程**：数据准备 → 数据链创建 → 预处理任务创建 → 任务执行 → 结果输出
- **数据标注流程**：数据准备 → 标注执行 → 结果导出

#### 业务规则

- API Key用户最多创建3个、租户最多创建1000个
- 租户下接入点ID唯一，保留Claude  --  claude\*,    OpenAI--gpt\*,Qwen -- qwen\*,Gemini--gemini\*, Kimi --kimi\* ,Doubao -- doubao\*,ChatGLM--glm\*,    Minimax--minimax,DeepSeek-deepseek这些ID

**Sprint2优化项规则：**

- API Key绑定私有模型下线时需fallback处理
- /v1/models接口不返回已下线模型
- model_id重复上架会导致计费错乱，需避免
- Key路由隔离：确保key1失效不会串到key2
- 兼容端点类型明确
- 模型下架后支持修改modelId

#### 业务指标

- **总请求数**：模型调用总次数
- **总 Token 数**：输入 Token + 输出 Token
- **总消费（点）**：模型调用消耗的点数
- **平均耗时**：模型响应平均时间
- **P95 耗时**：95 分位响应时间
- **P99 耗时**：99 分位响应时间
- **模型分布**：各模型的请求量、Token 量、消费量占比
- **API Key 消费分布**：各 API Key 的消费占比
- **端点分布**：各 API 端点的调用量分布

#### 业务角色

- **平台管理员**：运营租户下的管理员
- **模型开发者**：创建、训练、微调模型的用户
- **数据工程师**：管理数据集、数据预处理、数据标注的用户
- **业务集成者**：使用接入点进行业务系统集成的用户
- **评测人员**：执行模型评测任务的用户

## 9. 高阶衍生推理知识

### 9.1 链路类

#### 模型调用全链路

业务系统 → 接入点 → 模型服务 → 用量 消费结算

#### 模型训练全链路

数据准备 → 数据预处理 → 数据标注 → 算法选择 → 模型训练 → 模型评测 → 模型上架 → 模型广场

#### 模型微调全链路

基座模型选择 → 训练数据准备 → 微调配置 → 微调执行 → 模型评测 → 模型发布 → 接入点关联

### 9.2 影响类

#### 模型升级影响

- 直接影响：模型版本变更、性能变化
- 间接影响：通过接入点隔离，业务系统无感知
- 传导路径：模型库更新 → 接入点切换 → 业务系统无感

#### 数据质量影响

- 直接影响：模型训练效果、评测结果准确性
- 间接影响：模型上线后的业务效果
- 传导路径：数据质量 → 模型质量 → 业务效果

### 9.3 风险类

#### API Key 泄露风险

- 风险等级：高
- 影响范围：资金损失、数据泄露
- 防控措施：定期轮换、权限最小化、脱敏展示

#### 模型服务可用性风险

- 风险等级：「待完善」
- 影响范围：「待完善」
- 防控措施：「待完善」

### 9.4 决策类

#### 模型选型决策

- 决策依据：模型类型、上下文长度、语言支持、适用场景、供应商
- 决策流程：需求分析 → 模型筛选 → 体验验证 → 评测对比 → 接入点创建

#### 成本优化决策

- 决策依据：用量统计、消费分析、模型性能
- 优化方向：模型选型优化、调用量管控、接入点复用
